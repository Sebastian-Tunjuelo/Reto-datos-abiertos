## Agente

- **role**: Desarrollador frontend React/Next.js.
- **goal**: Implementar el botón de descarga de PDF que llama a `GET /reporte/{municipio}/{cultivo}?formato=pdf`,
  maneja el estado de carga y descarga el archivo en el navegador.
- **backstory**: El endpoint ya funciona y retorna bytes de PDF. El componente debe descargar el
  archivo sin abrir una nueva pestaña, usando la API de Blob del navegador.

## Objetivo

Implementar `BotonReportePDF` como componente React que descarga el reporte PDF del municipio
y cultivo seleccionados, con estado de carga, manejo de errores y nombre de archivo descriptivo.

## Alcance

### Incluye

- Componente `BotonReportePDF` en `frontend/src/components/chat/BotonReportePDF.tsx`.
- Botón con icono `FileDown` de lucide-react y texto "Generar reporte PDF".
- Estado de carga: spinner + texto "Generando..." mientras descarga.
- Descarga del PDF usando `URL.createObjectURL(blob)` + `<a download>` programático.
- Nombre del archivo: `reporte_{municipio}_{cultivo}_{fecha}.pdf` (sin tildes, sin espacios).
- Manejo de error: toast o mensaje inline si la descarga falla.
- Deshabilitar el botón si `municipio` o `cultivo` están vacíos.
- Tooltip con el texto "Descargar reporte institucional en PDF" cuando el botón está habilitado.

### No incluye

- Vista previa del PDF en el navegador.
- Generación del PDF en el frontend — el backend lo genera.
- Descarga del reporte en formato texto.
- Integración con el historial de chat.

## Entradas

| Prop          | Tipo      | Descripción                                                                                    |
| ------------- | --------- | ---------------------------------------------------------------------------------------------- |
| `municipio`   | `string`  | Nombre display del municipio (para el nombre del archivo).                                     |
| `codigoDane`  | `string`  | Código DANE de 5 dígitos (para la URL del endpoint).                                           |
| `cultivo`     | `string`  | Cultivo seleccionado.                                                                           |
| `apiBase`     | `string`  | Base URL del backend.                                                                           |
| `disabled`    | `boolean` | Opcional. Si `true`, deshabilitar el botón independientemente del contexto.                    |

## Lógica / Proceso sugerido

1. Al hacer clic, verificar que `codigoDane` y `cultivo` no estén vacíos.
2. Poner `isLoading = true` y deshabilitar el botón.
3. Llamar `GET {apiBase}/reporte/{codigoDane}/{encodeURIComponent(cultivo)}?formato=pdf`.
4. Si la respuesta es exitosa (`Content-Type: application/pdf`):
   - Convertir a `Blob`.
   - Crear `URL.createObjectURL(blob)`.
   - Crear un elemento `<a>` temporal con `href` y `download = reporte_{municipio_normalizado}_{cultivo_normalizado}_{YYYY-MM-DD}.pdf`.
   - Hacer clic programático y revocar la URL.
5. Si la respuesta falla:
   - Mostrar mensaje de error inline bajo el botón.
6. Poner `isLoading = false`.

### Normalización del nombre de archivo

```typescript
function normalizarNombreArchivo(texto: string): string {
  return texto
    .toLowerCase()
    .normalize("NFD")
    .replace(/[\u0300-\u036f]/g, "")  // quitar tildes
    .replace(/\s+/g, "_")
    .replace(/[^a-z0-9_]/g, "");
}
// Ejemplo: "Chaparral" → "chaparral", "Café" → "cafe"
// Nombre final: "reporte_chaparral_cafe_2025-05-29.pdf"
```

## Manejo de errores

| Situación                                              | Comportamiento                                                                                                    |
| ------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------- |
| `codigoDane` o `cultivo` vacíos                        | Botón deshabilitado con tooltip "Selecciona municipio y cultivo"                                                  |
| `GET /reporte` retorna error de red                    | Mensaje inline: "Error de conexión. Verifica que la API esté disponible."                                         |
| `GET /reporte` retorna 404                             | Mensaje inline: "No hay reporte disponible para {municipio}/{cultivo}."                                           |
| `GET /reporte` retorna 500                             | Mensaje inline: "Error al generar el reporte. Intenta de nuevo."                                                  |
| `Content-Type` no es `application/pdf`                 | Mensaje inline: "El servidor no retornó un PDF válido."                                                           |

## Salida esperada

- Botón renderizable sin errores de TypeScript.
- Al hacer clic con contexto válido, el navegador descarga un archivo `.pdf`.
- El nombre del archivo sigue el patrón `reporte_{municipio}_{cultivo}_{fecha}.pdf`.
- El botón muestra spinner durante la descarga.
- Los errores se muestran inline bajo el botón, no como alert del navegador.

## Caso feliz

- Dado que `codigoDane="73168"`, `cultivo="Café"` y el backend está disponible.
- Cuando el usuario hace clic en "Generar reporte PDF".
- Entonces el navegador descarga `reporte_chaparral_cafe_2025-05-29.pdf` sin abrir nueva pestaña.

## Validación

- Dado que `codigoDane` está vacío.
- Cuando se renderiza el componente.
- Entonces el botón está deshabilitado y muestra el tooltip "Selecciona municipio y cultivo".

- Dado que `GET /reporte` retorna 404.
- Cuando el usuario hace clic.
- Entonces se muestra el mensaje de error inline y el botón vuelve a estar habilitado.

## Criterios de aceptación

1. El componente renderiza sin errores de TypeScript.
2. El botón está deshabilitado cuando `codigoDane` o `cultivo` están vacíos.
3. Al hacer clic, el navegador descarga el PDF sin abrir nueva pestaña.
4. El nombre del archivo no tiene tildes ni espacios.
5. El spinner aparece durante la descarga y desaparece al completar.
6. Los errores se muestran inline, no como `alert()`.
7. El botón vuelve a estar habilitado después de un error.
8. La URL del blob se revoca después de la descarga para evitar memory leaks.
