## Agente

- **role**: Desarrollador frontend Next.js 14 App Router.
- **goal**: Crear la ruta `/asistente` con el layout de la pantalla de chat, el selector de contexto
  (municipio, cultivo, tono) y los placeholders para los componentes de chat y reporte.
- **backstory**: El frontend ya tiene F1/F2/F3 con patrones establecidos: header sticky, `"use client"`,
  shadcn/ui, lucide-react. F4.1 sigue exactamente esos patrones.

## Objetivo

Crear `frontend/src/app/asistente/page.tsx` con el layout completo de la pantalla de asistente:
header con navegación, selector de contexto (municipio/cultivo/tono) y área principal donde
se montarán `ChatAsistente` y `BotonReportePDF`.

## Alcance

### Incluye

- Archivo `frontend/src/app/asistente/page.tsx` con directiva `"use client"`.
- Header sticky con botón "← Mapa" y título "Asistente IA".
- Componente `SelectorContexto` en `frontend/src/components/chat/SelectorContexto.tsx`:
  - Select de municipio (carga desde `GET /municipios`).
  - Select de cultivo (carga desde `GET /cultivos/{municipio}` al cambiar municipio).
  - Select de tono: `campesino` / `institucional`.
- Lectura de `?municipio=...&cultivo=...` desde `useSearchParams` para preseleccionar el contexto.
- Área principal dividida en dos columnas en desktop (chat | reporte) y una columna en mobile.
- Placeholders tipados para `ChatAsistente` y `BotonReportePDF` (se implementan en F4.2 y F4.3).
- Banner de advertencia si `NEXT_PUBLIC_API_BASE_URL` no está configurado.

### No incluye

- Lógica de envío de mensajes al chat — eso es F4.2.
- Lógica de descarga de PDF — eso es F4.3.
- Persistencia del historial de mensajes.
- Autenticación ni control de acceso.

## Entradas

- `searchParams.municipio`: código DANE o nombre del municipio (opcional, desde URL).
- `searchParams.cultivo`: nombre del cultivo (opcional, desde URL).
- `GET /municipios` → `{ municipios: [{ codigo_dane, municipio, departamento }] }`.
- `GET /cultivos/{municipio}` → `{ cultivos: string[] }`.

## Inputs esperados

| Input                        | Tipo     | Regla                                                                                                    |
| ---------------------------- | -------- | -------------------------------------------------------------------------------------------------------- |
| `searchParams.municipio`     | `string` | Opcional. Si presente, preseleccionar en el selector. Puede ser código DANE o nombre.                    |
| `searchParams.cultivo`       | `string` | Opcional. Si presente, preseleccionar en el selector de cultivo.                                         |
| `NEXT_PUBLIC_API_BASE_URL`   | `string` | Si vacío o no definido, mostrar banner de advertencia y deshabilitar los selectores.                     |

## Lógica / Proceso sugerido

1. Leer `searchParams` con `useSearchParams()` para obtener `municipio` y `cultivo` iniciales.
2. Al montar, llamar `GET /municipios` para poblar el selector de municipios.
   - Si falla, usar lista vacía y mostrar error inline.
3. Cuando el municipio cambia, llamar `GET /cultivos/{municipio}` para poblar el selector de cultivos.
   - Si falla, usar `["Café", "Cacao", "Maíz"]` como fallback.
4. Si `searchParams.municipio` está presente, preseleccionar el municipio correspondiente.
5. Si `searchParams.cultivo` está presente, preseleccionar el cultivo correspondiente.
6. Pasar `{ municipio, cultivo, tono }` como props a `ChatAsistente` y `BotonReportePDF`.

## Manejo de errores

| Situación                                              | Comportamiento                                                                                                    |
| ------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------- |
| `NEXT_PUBLIC_API_BASE_URL` no configurado              | Banner amarillo: "API no configurada. Configura NEXT_PUBLIC_API_BASE_URL en .env.local"                           |
| `GET /municipios` falla                                | Selector de municipio vacío con mensaje "No se pudieron cargar los municipios"                                    |
| `GET /cultivos/{municipio}` falla                      | Usar fallback `["Café", "Cacao", "Maíz"]` sin mostrar error                                                       |
| `searchParams.municipio` no coincide con ningún municipio | Ignorar el parámetro y dejar el selector sin selección inicial                                                 |

## Salida esperada

- Ruta `/asistente` accesible y renderizable.
- Selector de municipio poblado con los 15 municipios MVP.
- Selector de cultivo actualizado al cambiar municipio.
- Selector de tono con opciones "Lenguaje campesino" e "Institucional (UMATA)".
- Si se navega desde `/municipio/73168/Café`, el contexto se preselecciona automáticamente.

## Caso feliz

- Dado que el usuario navega a `/asistente?municipio=73168&cultivo=Café`.
- Cuando la página carga.
- Entonces el selector de municipio muestra "Chaparral" preseleccionado, el selector de cultivo muestra "Café" preseleccionado, y el tono por defecto es "campesino".

## Validación

- Dado que `NEXT_PUBLIC_API_BASE_URL` está vacío.
- Cuando la página carga.
- Entonces se muestra el banner de advertencia y los selectores están deshabilitados.

## Criterios de aceptación

1. La ruta `/asistente` renderiza sin errores de TypeScript.
2. El selector de municipio carga los datos desde `GET /municipios`.
3. El selector de cultivo se actualiza al cambiar el municipio.
4. Los parámetros de URL preseleccionan el contexto correctamente.

6. El header tiene el mismo estilo que F1/F2/F3 (sticky, backdrop-blur, border-b).
7. No hay errores de hidratación SSR/CSR.
