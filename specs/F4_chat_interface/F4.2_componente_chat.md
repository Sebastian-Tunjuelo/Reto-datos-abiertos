## Agente

- **role**: Desarrollador frontend React/Next.js.
- **goal**: Implementar el componente de chat con historial de mensajes, input de texto, envío
  al endpoint `POST /chat` y manejo de estados de carga, error y fallback.
- **backstory**: El backend ya tiene `POST /chat` funcionando con Google Gemini y fallback local.
  El componente debe ser tolerante a respuestas lentas (el LLM puede tardar 3-8 segundos) y
  mostrar un indicador de escritura mientras espera.

## Objetivo

Implementar `ChatAsistente` como componente React que gestiona el historial de mensajes,
envía preguntas al endpoint `POST /chat` con el contexto del selector, y muestra las respuestas
con las fuentes citadas.

## Alcance

### Incluye

- Componente `ChatAsistente` en `frontend/src/components/chat/ChatAsistente.tsx`.
- Estado local del historial: array de `{ role: 'user' | 'assistant', content: string, fuentes?: string[], timestamp: Date }`.
- Input de texto con `Textarea` de shadcn/ui, envío con Enter (Shift+Enter para nueva línea) y botón Send.
- Indicador de "escribiendo..." (tres puntos animados) mientras espera la respuesta del LLM.
- Renderizado del historial con burbujas diferenciadas: usuario (derecha, color primario) y asistente (izquierda, color card).
- Sección de fuentes colapsable bajo cada respuesta del asistente (si `fuentes` no está vacío).
- Mensaje de bienvenida inicial cuando el historial está vacío.
- Scroll automático al último mensaje al recibir respuesta.
- Manejo de error: si `POST /chat` falla, mostrar mensaje de error inline en el historial.
- Botón "Limpiar conversación" para resetear el historial.

### No incluye

- Persistencia del historial en localStorage ni en base de datos.
- Streaming de la respuesta (la API retorna la respuesta completa).
- Markdown rendering avanzado — texto plano con saltos de línea es suficiente.
- Autenticación ni rate limiting en el frontend.
- Integración con el botón de PDF — eso es F4.3.

## Lógica / Proceso sugerido

### Envío de mensaje

1. El usuario escribe en el `Textarea` y presiona Enter o el botón Send.
2. Si el input está vacío o solo tiene espacios, no enviar.
3. Agregar el mensaje del usuario al historial con `role: 'user'`.
4. Limpiar el input y poner `isLoading = true`.
5. Llamar `POST /chat` con `{ pregunta, municipio: codigoDane, cultivo, año: anio, tono }`.
6. Si la respuesta es exitosa:
   - Agregar la respuesta al historial con `role: 'assistant'`, `content: data.respuesta`, `fuentes: data.fuentes`.
7. Si la respuesta falla (error de red o status >= 400):
   - Agregar mensaje de error al historial: `role: 'assistant'`, `content: "Lo siento, no pude procesar tu pregunta. Intenta de nuevo."`.
8. Poner `isLoading = false`.
9. Hacer scroll al último mensaje.

### Mensaje de bienvenida

Cuando el historial está vacío, mostrar:
```
Hola. Soy el asistente de SiembraSegura IA.
Puedes preguntarme sobre el riesgo agrícola, rendimiento esperado o recomendaciones
para {municipio} · {cultivo}.
```
Si no hay municipio/cultivo seleccionado: "Selecciona un municipio y cultivo para comenzar."

## Manejo de errores

| Situación                                              | Comportamiento                                                                                                    |
| ------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------- |
| `POST /chat` retorna error de red                      | Agregar mensaje de error al historial con `isError: true`, estilo rojo suave                                      |
| `POST /chat` retorna status >= 400                     | Agregar mensaje de error al historial con el detalle del error si está disponible                                 |
| `POST /chat` tarda más de 30 segundos                  | Mostrar mensaje de timeout en el historial                                                                        |
| Input vacío al enviar                                  | No enviar, no mostrar error                                                                                       |
| `municipio` o `cultivo` vacíos                         | Deshabilitar el input con placeholder "Selecciona municipio y cultivo para comenzar"                              |

## Salida esperada

- Componente renderizable sin errores de TypeScript.
- El historial muestra mensajes del usuario y del asistente con estilos diferenciados.
- El indicador de escritura aparece mientras espera la respuesta.
- Las fuentes se muestran como badges colapsables bajo la respuesta.
- El scroll automático funciona al recibir cada respuesta.

## Caso feliz

- Dado que el usuario escribe "¿Cuál es el riesgo para café en Chaparral?" y presiona Enter.
- Cuando `POST /chat` retorna `{ respuesta: "El riesgo es Alto...", fuentes: ["predicciones_con_explicacion.parquet"] }`.
- Entonces el historial muestra el mensaje del usuario y la respuesta del asistente, con las fuentes como badges.

## Validación

- Dado que el usuario presiona Enter con el input vacío.
- Cuando se intenta enviar.
- Entonces no se agrega ningún mensaje al historial y no se llama a la API.

- Dado que `POST /chat` falla con error de red.
- Cuando se envía un mensaje.
- Entonces se agrega un mensaje de error al historial con estilo diferenciado y el input queda habilitado.

## Criterios de aceptación

1. El componente renderiza sin errores de TypeScript.
2. El historial muestra mensajes con burbujas diferenciadas por rol.
3. El indicador de escritura aparece durante la carga y desaparece al recibir respuesta.
4. Enter envía el mensaje; Shift+Enter agrega nueva línea.
5. El scroll automático lleva al último mensaje.
6. Las fuentes se muestran como badges bajo la respuesta del asistente.
7. El input se deshabilita mientras hay una petición en curso.
8. El botón "Limpiar conversación" resetea el historial al mensaje de bienvenida.
9. El componente es accesible: `aria-label` en el textarea, roles semánticos en los mensajes.
