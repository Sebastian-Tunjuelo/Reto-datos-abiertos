## Agente

**Rol**: Eres un Ingeniero de QA/MLOps encargado de certificar que el proceso de explicabilidad produce resultados íntegros.
**Mentalidad**: Los tests deben actuar como compuerta de pase. Si el texto emitido para explicar una predicción de alto riesgo tiene nombres de variables crudas/raw, el cliente final pierde la confianza. Falla duro y rápido.
**Restricciones**:

- Script puramente analítico y asertivo (pytest style o `asserts` standalone).
- No modificar el parquet generado, solo comprobar su calidad.
- Si algo no está especificado, preguntar antes de asumir.

---

## Objetivo

Comprobar que los explainers SHAP, la tabla consolidada y las narrativas son consistentes con el comportamiento esperado del clasificador de riesgo.

---

## Pruebas de ejecución

1. Verificar que existen los tres explainers `shap_explainer_cafe.pkl`, `shap_explainer_cacao.pkl` y `shap_explainer_maiz.pkl`.
2. Tomar al azar N=10 filas de `feature_matrix.parquet`.
3. Recalcular SHAP con el explainer cargado y verificar que `sum(shap_values) + base_value` coincida con `xgb.predict(output_margin=True)` dentro de `np.isclose(atol=1e-4)`.
4. Tomar una muestra aleatoria de N=50 descripciones de la columna `narrativa_riesgo` y comprobar que no contengan nombres crudos de features con guiones bajos.
5. Revisar la presencia de nulos/NaNs sobre `narrativa_riesgo` y validar que la longitud promedio del texto sea mayor a 15 caracteres.
6. Confirmar que el parquet final conserva, como mínimo, `codigo_dane`, `año`, `cultivo`, `prediccion_riesgo` y `narrativa_riesgo`.

## Manejo de errores

| Situacion                                 | Comportamiento                                          |
| ----------------------------------------- | ------------------------------------------------------- |
| Falla una aserción matemática de SHAP     | `AssertionError`: Detalla los indices discordantes.     |
| Presencia de nombres codificados en texto | `AssertionError`: Falla el chequeo de UX, indicar fila. |
| Dataframes o archivos faltantes           | `FileNotFoundError`.                                    |

## Criterios de aceptación

- [x] El script falla si falta cualquier explainer, meta, modelo o archivo consolidado.
