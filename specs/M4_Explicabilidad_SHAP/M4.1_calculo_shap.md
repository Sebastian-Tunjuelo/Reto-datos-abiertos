## Objetivo

Instanciar un `TreeExplainer` por cultivo, calcular los valores SHAP sobre la matriz de features usada por el clasificador de riesgo y guardar el explainer listo para inferencia y validación.

---

## Inputs

| Archivo                             | Descripción / Columnas minimas requeridas                                                |
| ----------------------------------- | ---------------------------------------------------------------------------------------- |
| `data/feature_matrix.parquet`       | Dataset base de features. Mismas columnas que M1/M2/M3.                                  |
| `models/xgb_classifier_*.pkl`       | Modelos XGBoost de predicción de riesgo entrenados en M3.                                |
| `models/xgb_classifier_*_meta.json` | Metadatos indicando las features en el orden exacto en el que entraron al entrenamiento. |

---

## Lógica de cálculo y extracción

1. **Cargar información**: Por cada cultivo (Café, Cacao, Maíz), cargar el json de metadatos para conocer las features utilizadas, el modelo XGBoost y filtrar la matriz de features `data/feature_matrix.parquet`.
2. **Validar features**: Verificar que todas las columnas declaradas en `feature_cols` existan en la matriz base.
3. **Reordenar inputs**: Filtrar y reordenar `X` exactamente como en el entrenamiento.
4. **Instanciar explainer**: Crear `shap.TreeExplainer(model)` para cada cultivo.
5. **Calcular valores SHAP**: Ejecutar `explainer.shap_values(X)` sobre el conjunto objetivo.
6. **Capturar base value**: Obtener `expected_value` o `base_value` en el mismo espacio raw del clasificador.
7. **Persistir explainer**: Serializar con `pickle` a `models/shap_explainer_{cultivo}.pkl`.
8. **Retornar data en memoria**: Devolver un diccionario con explainers, SHAP values y feature order por cultivo.

---

## Output

- Archivos generados: `models/shap_explainer_cafe.pkl`, `models/shap_explainer_cacao.pkl`, `models/shap_explainer_maiz.pkl`.
- Salida en memoria: diccionario con explainers, SHAP values y `expected_value` / `base_value` en espacio raw.

---

## Manejo de errores

| Situacion                                   | Comportamiento                                           |
| ------------------------------------------- | -------------------------------------------------------- |
| Archivos de modelo o metadatos no existen   | `FileNotFoundError` detallando el cultivo faltante.      |
| Features listadas en JSON no están en datos | `KeyError` o `ValueError` detallando columnas faltantes. |
| `shap_values` y `feature_cols` no coinciden | `ValueError` con el mismatch de dimensiones.             |
| Suma total de SHAP y base_value difiere     | Lanzar `AssertionError` y detener la operación.          |
| Instanciación lenta (>= 5 segs)             | Warning de lentitud.                                     |
