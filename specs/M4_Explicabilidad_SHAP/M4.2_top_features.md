## Agente

**Rol**: Eres un desarrollador Backend/Data trabajando en procesamiento e interpretación post-modelo.
**Mentalidad**: El usuario final necesita una explicación breve y clara. Prioriza el mapeo humano y el orden por impacto absoluto.
**Restricciones**:

- Siempre extraer exactamente `top_n` features cuando existan contribuciones no nulas.
- Mantener la dirección e impacto (agravante o mitigante) intacto.
- Si una variable no tiene mapeo humano configurado, debe generar una advertencia y usar el nombre raw o un fallback legible, no fallar catastróficamente.
- No perder la referencia al valor original observado en la fila.
- Si algo no está especificado, preguntar antes de asumir.

---

## Objetivo

Dado un vector SHAP para una predicción concreta, ordenar las variables por magnitud absoluta, separar si aumentan o disminuyen el riesgo y mapear cada feature técnica a un nombre amigable para su consumo.

---

## Inputs

| Archivo / Objeto  | Descripción / Detalles                                          |
| ----------------- | --------------------------------------------------------------- |
| `shap_values_row` | Vector 1D con los valores SHAP de una predicción.               |
| `feature_names`   | Lista ordenada con los nombres de las columnas.                 |
| `original_row`    | `dict` o `Series` con los valores originales de la observación. |
| `feature_mapping` | Diccionario de nombres técnicos a nombres amigables.            |
| `top_n`           | Cantidad de factores a devolver; por defecto `5`.               |

---

## Lógica de extracción

1. Validar que `shap_values_row` y `feature_names` tengan la misma longitud.
2. Ordenar por `abs(shap_value)` en orden descendente.
3. Seleccionar exactamente los `top_n` elementos con mayor magnitud absoluta.
4. Determinar la dirección: `shap_value > 0` aumenta riesgo; `shap_value < 0` disminuye riesgo.
5. Obtener el valor original correspondiente a cada feature.
6. Buscar el nombre amigable en `feature_mapping`.
7. Si no existe mapeo, usar `feature_name.replace("_", " ").title()` y emitir warning.
8. Retornar una lista de diccionarios JSON-like lista para serializar.

---

## Output

Una lista de diccionarios, o una nueva columna serializada en parquet si es batch, generada por cada fila y estructurada así:

```json
[
  {
    "feature_id": "anomalia_prec",
    "nombre_amigable": "Anomalía de lluvias",
    "shap_value": 0.45,
    "direccion": "Aumenta riesgo",
    "valor_original": -1.2
  }
]
```

---

## Manejo de errores

| Situacion                                        | Comportamiento                                                            |
| ------------------------------------------------ | ------------------------------------------------------------------------- |
| Dimensión de SHAP distinta al número de features | `ValueError` con mensaje de mismatch                                      |
| Falta mapeo humano para una feature              | Warning y fallback automático                                             |
| Todas las variables tienen SHAP = 0              | Retornar lista vacía o una estructura equivalente sin factores dominantes |

## Criterios de aceptación

- [x] La función retorna exactamente `top_n` factores cuando existen contribuciones no nulas.
- [x] Se mantiene el signo de cada impacto.
- [x] Las features sin mapeo usan fallback y generan warning.
