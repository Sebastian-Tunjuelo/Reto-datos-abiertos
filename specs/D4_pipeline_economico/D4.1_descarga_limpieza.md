### Formatos de fecha observados
El campo `fecha` puede venir en varios formatos según el año de carga del registro:
| Formato | Ejemplo | Frecuencia |
|---------|---------|------------|
| `'YYYY-MM-DDT00:00:00.000'` | `'2015-03-01T00:00:00.000'` | Más común |
| `'YYYY-MM-DD'` | `'2015-03-01'` | Ocasional |
| `'MM/YYYY'` | `'03/2015'` | Registros antiguos |

Usar `pd.to_datetime(fecha, errors='coerce')` para parsear todos los formatos. Extraer `año` y `mes` del resultado. Construir `fecha` canónica como `f"{año:04d}-{mes:02d}"`.

## Lógica de transformación
```
1. Llamar fetch_all(dataset_id='gwbi-fnzs') — sin filtros, descargar completo
2. Convertir lista de dicts → pd.DataFrame
3. Verificar que las columnas esperadas existen; loggear las columnas reales si hay discrepancia
4. Renombrar columnas según mapeo (total_fertilizantes → fertilizantes, etc.)
5. Parsear fecha:
   a. pd.to_datetime(df['fecha'], errors='coerce') → fecha_dt
   b. Loggear cuántas fechas no pudieron parsearse (fecha_dt.isna().sum())
   c. Eliminar filas con fecha_dt nula
   d. año = fecha_dt.dt.year.astype('Int64')
   e. mes = fecha_dt.dt.month.astype('Int64')
   f. fecha = fecha_dt.dt.strftime('%Y-%m')  # formato canónico 'YYYY-MM'
6. Convertir columnas numéricas con pd.to_numeric(..., errors='coerce'):
   indice_total, fertilizantes, plaguicidas, urea, dap, kcl
7. Filtrar: conservar solo filas con año entre 2007 y 2024 inclusive
   Loggear cuántas filas quedan fuera del rango
8. Eliminar duplicados por (año, mes)
   Loggear cuántos duplicados se eliminaron
9. Ordenar por (año, mes) ascendente
10. Retornar DataFrame con columnas en orden del esquema mensual
```

## Notas de implementación

### Descarga sin filtros
El dataset tiene ~200 registros — menos que el límite de una sola llamada Socrata (5.000). Usar `fetch_all()` sin parámetros `where` ni `select`. Si en el futuro el dataset crece, `fetch_all()` maneja la paginación automáticamente.

### Nombres de columnas en API
La API usa `indice_total` y prefijos `total_` para los agregados. La implementación debe buscar estos nombres exactos. Si el dataset cambia, el código debe fallar con un `KeyError` claro — no silenciosamente. Esto es intencional: preferimos un error explícito a datos incorrectos.

Actualizacion 2026-05-17: el dataset expone `total_fertilizantes`, `total_plaguicidas`, `urea_46`, `dap_18_46`, `kcl_0_0_60` y `indice_total`.

### Tipo de `año` y `mes`
Se usa `Int64` (nullable integer de pandas) para consistencia con el resto del proyecto. En la práctica todos los registros tienen fecha válida, pero la robustez es preferible.

### Valores nulos en índices
Es normal que algunos meses tengan `NaN` en sub-índices específicos (ej. `kcl` puede no reportarse todos los meses). No eliminar estas filas — el `NaN` es información válida.

## Manejo de errores
| Situación | Comportamiento |
|-----------|---------------|
| Fecha no parseable | Eliminar fila; loggear cuántas y qué valores |
| Columna numérica no parseable | → `NaN`; no eliminar la fila |
| Columna API con nombre inesperado | Loggear las columnas reales; lanzar `KeyError` descriptivo |
| Año fuera de rango 2007–2024 | Filtrar y loggear |
| Duplicados por (año, mes) | Eliminar y loggear |
| DataFrame vacío tras limpieza | Lanzar `ValueError("Agroinsumos: sin registros tras limpieza")` |
| Error HTTP de Socrata | `fetch_all()` reintenta 3 veces; si falla, propagar excepción |
