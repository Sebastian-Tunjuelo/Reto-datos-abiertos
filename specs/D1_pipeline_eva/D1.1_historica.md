## Objetivo
Descargar los registros de EVA 2007-2018 para los 15 municipios del MVP y los 3 cultivos, limpiarlos y retornar un DataFrame con el esquema unificado del proyecto.

| Campo | Valor |

### Columnas reales en la API
| Columna API | Tipo | Descripción |
|-------------|------|-------------|
| `c_d_mun` | string | Código DANE (puede venir sin cero inicial, ej. `'5036'`) |
| `municipio` | string | Nombre en MAYÚSCULAS |
| `departamento` | string | Nombre en MAYÚSCULAS |
| `cultivo` | string | `'CAFÉ'`, `'CACAO'`, `'MAÍZ'` |
| `a_o` | string | Año: `'2007'`..`'2018'` |
| `periodo` | string | `'2015A'`, `'2015B'` o `'2015'` |
| `rendimiento_t_ha` | string | t/ha (puede ser vacío o `'0'`) |
| `rea_sembrada_ha` | string | ha |
| `rea_cosechada_ha` | string | ha |
| `producci_n_t` | string | toneladas |
| `ciclo_de_cultivo` | string | `'PERMANENTE'` o `'TRANSITORIO'` |

### Filtro SoQL
```
$where=cultivo IN ('CAFÉ','CACAO','MAÍZ') AND c_d_mun IN ('73001','73168','41001','41298','41551','68689','68615','05036','05030','17541','17524','50001','19256','19418','20001')
```
> ⚠️ El filtro por `c_d_mun` puede no capturar registros con código sin cero inicial (ej. `'5036'`). Estrategia: incluir ambas formas en el `IN` o filtrar en Python después de normalizar.

## Lógica de transformación
```
1. Llamar fetch_all(dataset_id='2pnw-mmge', where=<filtro SoQL>)
2. Convertir lista de dicts → pd.DataFrame
3. Renombrar columnas según mapeo
4. normalize_dane_code(codigo_dane) → str 5 dígitos
5. Filtrar: conservar solo filas donde codigo_dane esté en MVP_CODIGOS
   (captura los que llegaron sin cero inicial)
6. normalize_cultivo(cultivo) → 'Café'|'Cacao'|'Maíz'|None
   Descartar filas donde cultivo sea None; loggear cuántas
7. normalize_title_case(municipio) y normalize_title_case(departamento)
8. año = pd.to_numeric(año, errors='coerce').astype('Int64')
9. rendimiento = pd.to_numeric(rendimiento, errors='coerce')
   rendimiento donde valor == 0.0 → None
   rendimiento fuera de RENDIMIENTO_RANGOS[cultivo] → None
10. area_sembrada, area_cosechada, produccion = pd.to_numeric(..., errors='coerce')
11. ciclo: conservar 'PERMANENTE'|'TRANSITORIO'; cualquier otro valor → None
12. fuente = 'historica'
13. Eliminar duplicados por (codigo_dane, cultivo, año, periodo, fuente)
    Loggear cuántos duplicados se eliminaron
14. Retornar DataFrame con columnas en orden del esquema unificado
```

## Manejo de errores
| Situación | Comportamiento |
|-----------|---------------|
| Municipio sin registros en la fuente | Warning: `"[D1.1] Sin datos para {municipio} en EVA histórica"` |
| `rendimiento` vacío o `'0'` | → `None` (cero no es válido agronómicamente) |
| `rendimiento` fuera de rango | → `None`; loggear municipio y cultivo afectados |
| Cultivo no reconocido (ej. `'MAIZ AMARILLO'`) | Descartar fila; loggear valor inesperado |
| Error HTTP de Socrata | `fetch_all()` reintenta 3 veces; si falla, propagar excepción |
| DataFrame vacío tras filtros | Lanzar `ValueError("EVA histórica: sin registros para el MVP")` |
