## Objetivo
Descargar los registros de EVA 2019-2024 para los 15 municipios del MVP y los 3 cultivos, limpiarlos y retornar un DataFrame con el esquema unificado del proyecto.

## Lógica de transformación
```
1. Llamar fetch_all(dataset_id='uejq-wxrr', where=<filtro SoQL>)
2. Convertir lista de dicts → pd.DataFrame
3. Renombrar columnas según mapeo
4. normalize_dane_code(codigo_dane) → str 5 dígitos (por consistencia)
5. Filtrar: conservar solo filas donde codigo_dane esté en MVP_CODIGOS
6. normalize_cultivo(cultivo) → 'Café'|'Cacao'|'Maíz'|None
   Descartar filas donde cultivo sea None; loggear cuántas
7. municipio y departamento ya vienen en Title Case; aplicar strip() por seguridad
8. año = pd.to_numeric(año, errors='coerce').astype('Int64')
9. rendimiento = pd.to_numeric(rendimiento, errors='coerce')
   rendimiento donde valor == 0.0 → None
   rendimiento fuera de RENDIMIENTO_RANGOS[cultivo] → None
10. area_sembrada, area_cosechada, produccion = pd.to_numeric(..., errors='coerce')
11. ciclo: conservar 'PERMANENTE'|'TRANSITORIO'; cualquier otro valor → None
12. fuente = 'reciente'
13. Eliminar duplicados por (codigo_dane, cultivo, año, periodo, fuente)
    Loggear cuántos duplicados se eliminaron
14. Retornar DataFrame con columnas en orden del esquema unificado
```

## Manejo de errores
| Situación | Comportamiento |
|-----------|---------------|
| Municipio sin registros en la fuente | Warning: `"[D1.2] Sin datos para {municipio} en EVA reciente"` |
| `rendimiento` vacío o `'0'` | → `None` |
| `rendimiento` fuera de rango | → `None`; loggear municipio y cultivo afectados |
| Cultivo no reconocido | Descartar fila; loggear valor inesperado |
| Error HTTP de Socrata | `fetch_all()` reintenta 3 veces; si falla, propagar excepción |
| DataFrame vacío tras filtros | Lanzar `ValueError("EVA reciente: sin registros para el MVP")` |

    Raises:
        ValueError: Si el resultado está vacío tras aplicar filtros.
        requests.RequestException: Si la API falla tras reintentos.
    """
