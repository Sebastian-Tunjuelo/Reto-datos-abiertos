## Consultas SoQL

### Temperatura — media anual y media de máximas
```sql
-- Consulta A: temperatura media anual
$select=municipio,
        date_trunc_y(fechaobservacion) AS año,
        avg(valorobservado) AS temp_media_c,
        max(valorobservado) AS temp_max_abs_c,
        count(*) AS n_observaciones_temp
$where=codigoestacion IN ('...lista de códigos de D2.1...')
       AND valorobservado IS NOT NULL
$group=municipio, date_trunc_y(fechaobservacion)
$order=municipio, año
```

> `avg()` y `max()` son funciones de agregación válidas en SoQL. `temp_max_abs_c` es el máximo absoluto del año — se usa como proxy de `temp_max_media_c` (media de máximas diarias), que requeriría una subconsulta no soportada en SoQL. Se documenta la limitación en el output.

### Humedad — media anual
```sql
$select=municipio,
        date_trunc_y(fechaobservacion) AS año,
        avg(valorobservado) AS hum_media_pct,
        count(*) AS n_observaciones_hum
$where=codigoestacion IN ('...lista de códigos de D2.1...')
       AND valorobservado IS NOT NULL
$group=municipio, date_trunc_y(fechaobservacion)
$order=municipio, año
```

## Lógica de transformación — `download_temperatura`
```
1. Recibir df_estaciones (output de D2.1)
   Extraer lista de codigo_estacion para los 15 municipios del MVP
   Si la lista está vacía → lanzar ValueError

2. Construir filtro SoQL con los códigos de estación
   Si len(codigos) > 500 → dividir en lotes de 500

3. Ejecutar Consulta A de temperatura:
   fetch_all(dataset_id='sbwg-7ju4', select=..., where=..., group=..., order=...)
   → df_temp

4. Convertir columna 'año': pd.to_datetime(año_str).dt.year → int

5. Convertir columnas numéricas con pd.to_numeric(..., errors='coerce'):
   temp_media_c, temp_max_abs_c, n_observaciones_temp

6. Filtrar valores físicamente imposibles:
   temp_media_c < 0 o > 45 → None; loggear municipio y año
   temp_max_abs_c < 0 o > 50 → None; loggear

7. normalize_title_case(municipio)

8. Cruzar municipio → codigo_dane usando get_codigo()

9. Filtrar: conservar solo filas donde codigo_dane esté en MVP_CODIGOS

10. Filtrar años: conservar solo 2007 ≤ año ≤ 2024

11. Renombrar temp_max_abs_c → temp_max_media_c
    (documentar en docstring que es el máximo absoluto, no la media de máximas)

12. Agregar n_estaciones_temp:
    Misma estrategia que D2.2 — consulta adicional con count(codigoestacion)

13. Loggear warning por cada municipio del MVP sin datos de temperatura
14. Retornar DataFrame con columnas:
    codigo_dane, municipio, año, temp_media_c, temp_max_media_c, n_estaciones_temp
```

## Lógica de transformación — `download_humedad`
```
1–9. Idéntico a download_temperatura, usando dataset 'uext-mhny'

10. Filtrar valores físicamente imposibles:
    hum_media_pct < 0 o > 100 → None; loggear municipio y año

11. Filtrar años: conservar solo 2007 ≤ año ≤ 2024

12. Loggear warning por cada municipio del MVP sin datos de humedad
13. Retornar DataFrame con columnas:
    codigo_dane, municipio, año, hum_media_pct
```

## Notas de implementación

### Función privada `_agregar_variable_climatica`
La lógica de los pasos 1–9 es idéntica entre `download_temperatura` y `download_humedad`. Se extrae a una función privada:
```python
def _agregar_variable_climatica(
    dataset_id: str,
    df_estaciones: pd.DataFrame,
    select: str,
    codigos_dane: list[str],
    prefijo_log: str,
) -> pd.DataFrame:
    """Descarga y agrega una variable climática por municipio y año."""
```
Cada función pública construye su `select` específico y delega a `_agregar_variable_climatica`.

### `temp_max_media_c` — limitación documentada
SoQL no soporta subconsultas, por lo que no es posible calcular la media de máximas diarias directamente. Se usa `max(valorobservado)` como proxy (máximo absoluto del año). Esta limitación se documenta en el docstring de la función y en el README del pipeline. Para el modelo predictivo, `temp_max_media_c` sigue siendo una señal útil de estrés térmico aunque no sea exactamente la media de máximas.

### Municipios con cobertura parcial
Igual que en D2.2: las filas faltantes quedan ausentes del DataFrame. D2.4 maneja los huecos.
