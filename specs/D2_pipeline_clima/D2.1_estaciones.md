### Filtro SoQL implementado
```
$where=municipio IN (
  'IBAGUE','CHAPARRAL','NEIVA','GARZON','PITALITO',
  'SAN VICENTE DE CHUCURI','RIONEGRO','ANORI','AMALFI',
  'PENSILVANIA','PALESTINA','VILLAVICENCIO',

## Lógica de transformación
```
1. Llamar fetch_all(dataset_id='hp9r-jxuu', where=<filtro SoQL>)
2. Convertir lista de dicts → pd.DataFrame
   Si DataFrame vacío → lanzar ValueError("Catálogo IDEAM: sin estaciones para el MVP")
3. Renombrar columnas según mapeo
4. strip() en columnas de texto: codigo_estacion, nombre_estacion, categoria, tecnologia, estado
5. normalize_title_case(municipio) → columna 'municipio'
6. normalize_title_case(departamento) → columna 'departamento'
7. latitud, longitud = pd.to_numeric(..., errors='coerce')
8. altitud_m = pd.to_numeric(..., errors='coerce')
9. Derivar codigo_dane:
   - Para cada fila: get_codigo(municipio_raw) → codigo_dane
   - Si no se encuentra → None; loggear estación sin código DANE
10. Filtrar: conservar solo filas donde codigo_dane esté en MVP_CODIGOS
    (red de seguridad ante variaciones de nombre no capturadas por SoQL)
11. Eliminar duplicados por codigo_estacion
12. Loggear resumen por municipio: cuántas estaciones totales y cuántas activas
13. Loggear warning por cada municipio del MVP sin ninguna estación
14. Retornar DataFrame con columnas en orden del esquema de salida
```

## Notas de implementación

### Variaciones de nombre de municipio en la API
El campo `municipio` en el catálogo IDEAM no siempre coincide exactamente con los nombres en `DANE_CODES`. Casos conocidos:
- `'SAN VICENTE DE CHUCURI'` puede aparecer como `'SAN VICENTE CHUCURI'` o `'SAN VICENTE DE CHUCURÍ'`.
- El filtro SoQL captura la forma más común; el paso 10 actúa como red de seguridad.
- Si un municipio queda sin estaciones por este motivo, el warning en el paso 13 lo hace visible.

### `municipio_raw` no se incluye en el output final
Se usa internamente en el paso 9 para el cruce con `DANE_CODES`. No se retorna en el DataFrame final para mantener el esquema limpio.

### Estaciones suspendidas
Se conservan en el output — D2.2 y D2.3 decidirán si usarlas o no según la cobertura temporal de cada una. Filtrar aquí sería prematuro.

## Manejo de errores
| Situación | Comportamiento |
|-----------|---------------|
| Municipio sin estaciones en la fuente | Warning: `"[D2.1] Sin estaciones para {municipio}"` |
| Estación sin código DANE derivable | Warning: `"[D2.1] Estación {codigo} sin código DANE"` — `codigo_dane = None` |
| Error HTTP de Socrata | `fetch_all()` reintenta 3 veces; si falla, propagar excepción |
| DataFrame vacío tras filtros | Lanzar `ValueError("Catálogo IDEAM: sin estaciones para el MVP")` |

## Interfaz de la función

def download_catalogo_estaciones(
    codigos_dane: list[str] | None = None,
) -> pd.DataFrame:
    """
    Descarga el catálogo de estaciones IDEAM para los municipios del MVP.

    Args:
        codigos_dane: Lista de códigos DANE a filtrar.
                      Si None, usa MVP_CODIGOS de shared/dane_codes.py.

    Returns:
        DataFrame con una fila por estación.
        Columnas: codigo_estacion, nombre_estacion, categoria, tecnologia,
                  estado, municipio, departamento, latitud, longitud,
                  altitud_m, codigo_dane.

    Raises:
        ValueError: Si el resultado está vacío tras aplicar filtros.
        requests.RequestException: Si la API falla tras reintentos.
    """
```
