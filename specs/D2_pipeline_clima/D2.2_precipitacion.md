## Agente

**Rol**: Eres un ingeniero de datos Python especializado en procesamiento de series temporales climáticas. Tu trabajo es descargar la precipitación de ~165 millones de registros IDEAM de forma eficiente usando agregaciones SoQL, y retornar una serie anual limpia por municipio.

- **Nunca** llamar `fetch_all()` sin `$group` en este dataset — son ~165M registros

- No usar `sodapy` — usar `shared/socrata_client.py` directamente
- No usar `inplace=True` en operaciones Pandas
- El filtro SoQL debe usar `municipio IN (...)` en MAYÚSCULAS
- Agregar siempre a nivel `(municipio, año)` — no retornar datos mensuales ni diarios

## Objetivo

Descargar la precipitación del dataset IDEAM `s54a-sgyg` para las estaciones de los 15 municipios del MVP, agregada por municipio y año, y retornar un DataFrame con acumulado anual, días secos y días de lluvia.

---

## Estrategia de filtrado — por municipio, no por codigoestacion

> ⚠️ **Cambio de arquitectura respecto al diseño original**

El diseño original filtraba por `codigoestacion IN (...)` usando la lista de estaciones de D2.1. En producción se descubrió que el servidor Socrata de `datos.gov.co` **no tiene índice sobre `codigoestacion`** en los datasets de observaciones. Cualquier consulta con ese filtro provoca un full-scan de 165M registros y hace timeout, incluso con 1 sola estación y 1 año de datos.

**Solución adoptada**: filtrar por `municipio IN (...)` en MAYÚSCULAS, que sí está indexado. Las consultas se dividen en **ventanas de 6 meses** (variable de entorno `IDEAM_MONTH_CHUNK`, default 6 meses) para acotar el volumen por llamada. Los resultados parciales se re-agregan localmente con pandas.

En la corrida que quedó guardada ya se obtuvo precipitación real para los municipios del MVP y se completó el parquet final junto con los intermedios.

**Consecuencia para D2.1**: el catálogo de estaciones sigue siendo útil como output de auditoría (qué estaciones existen por municipio), pero ya **no se usa como intermediario** para filtrar las consultas de observaciones. `df_estaciones` se acepta como parámetro por compatibilidad de interfaz.

### Consultas SoQL implementadas

Se hacen **tres consultas separadas**, cada una dividida en ventanas mensuales:

**Consulta A — Acumulado anual:**

```sql
$select=municipio,
        date_trunc_y(fechaobservacion) AS año,
        sum(valorobservado) AS prec_acum_mm,
        count(*) AS n_observaciones
$where=municipio IN ('IBAGUE','CHAPARRAL','NEIVA','GARZON','PITALITO',
                     'SAN VICENTE DE CHUCURI','RIONEGRO','ANORI','AMALFI',
                     'PENSILVANIA','PALESTINA','VILLAVICENCIO',
                     'EL TAMBO','MIRANDA','VALLEDUPAR')
       AND valorobservado IS NOT NULL
       AND fechaobservacion >= '2024-01-01T00:00:00.000'
       AND fechaobservacion < '2024-02-01T00:00:00.000'
$group=municipio, date_trunc_y(fechaobservacion)
$order=municipio, date_trunc_y(fechaobservacion)
-- (se repite para cada ventana mensual 2007-01 … 2024-12)
```

**Consulta B — Días secos y días de lluvia (umbral 1 mm):**

```sql
$select=municipio,
        date_trunc_y(fechaobservacion) AS año,
        count(CASE WHEN valorobservado < '1' THEN 1 END) AS prec_dias_secos,
        count(CASE WHEN valorobservado >= '1' THEN 1 END) AS prec_dias_lluvia
$where=municipio IN (...) AND valorobservado IS NOT NULL
       AND fechaobservacion >= '...' AND fechaobservacion < '...'
$group=municipio, date_trunc_y(fechaobservacion)
```

**Consulta C — n_estaciones_prec:**

```sql
$select=municipio,
        date_trunc_y(fechaobservacion) AS año,
        count(codigoestacion) AS n_estaciones_prec
$where=municipio IN (...) AND valorobservado IS NOT NULL
       AND fechaobservacion >= '...' AND fechaobservacion < '...'
$group=municipio, date_trunc_y(fechaobservacion)
```

> **Re-agregación local**: como cada ventana mensual devuelve resultados parciales por año, se re-agregan con `groupby(["municipio","año"]).agg(sum)` en pandas antes del merge final.

> **Nota sobre `date_trunc_y`**: La función de Socrata para truncar a año es `date_trunc_y(campo)`. El resultado viene como string ISO, ej. `'2015-01-01T00:00:00.000'`. Se extrae el año con `pd.to_datetime(...).dt.year`.

## Lógica de transformación

```
1. Recibir df_estaciones (output de D2.1) — validar que no esté vacío
   (no se usa para construir el filtro; se acepta por compatibilidad de interfaz)

2. Construir filtro SoQL con municipios en MAYÚSCULAS (no codigoestacion)
   Generar ventanas mensuales 2007-01 … 2024-12 (IDEAM_MONTH_CHUNK meses c/u)

3. Ejecutar Consulta A (acumulado) para cada ventana mensual:
   fetch_all(dataset_id='s54a-sgyg', select=..., where=..., group=..., order=...)
   Concatenar resultados → df_acum_raw

4. Re-agregar df_acum_raw por (municipio, año) con sum() en pandas
   → df_acum

5. Ejecutar Consulta B (días secos/lluvia) para cada ventana mensual
   Re-agregar por (municipio, año) con sum() → df_dias

6. Ejecutar Consulta C (n_estaciones_prec) para cada ventana mensual
   Re-agregar por (municipio, año) con sum() → df_nest

7. Convertir columna 'año' en todos los DataFrames:
   pd.to_datetime(año_str, errors='coerce').dt.year → int

8. Convertir columnas numéricas con pd.to_numeric(..., errors='coerce')

9. normalize_title_case(municipio) en todos los DataFrames

10. Cruzar municipio → codigo_dane usando get_codigo() de shared/dane_codes.py

11. Filtrar: conservar solo filas donde codigo_dane esté en codigos_objetivo

12. Hacer merge de df_acum, df_dias y df_nest por (codigo_dane, municipio, año)
    Usar how='outer' para no perder filas con datos parciales

13. Filtrar años fuera de rango: conservar solo YEAR_START ≤ año ≤ YEAR_END

14. Loggear warning por cada municipio del MVP sin datos de precipitación
15. Loggear resumen: registros por municipio, rango de años, % de NaN en prec_acum_mm

16. Retornar DataFrame con columnas:
    codigo_dane, municipio, año, prec_acum_mm, prec_dias_secos, prec_dias_lluvia, n_estaciones_prec
```

## Notas de implementación

### Estrategia de filtrado — por municipio, no por codigoestacion

El servidor Socrata de `datos.gov.co` no tiene índice sobre `codigoestacion` en el dataset `s54a-sgyg`. Filtrar por ese campo provoca un full-scan de 165M registros y hace timeout. Se filtra por `municipio` (MAYÚSCULAS) que sí está indexado.

### Ventanas mensuales

Con `IDEAM_MONTH_CHUNK=6` se generan 36 ventanas (18 años × 2 ventanas por año). Cada ventana hace una llamada `fetch_all()` con `$group`, nunca datos crudos. Los resultados se re-agregan localmente con `groupby().agg(sum)` antes del merge final.

### `n_estaciones_prec` — Consulta C

Se hace una tercera consulta con `count(codigoestacion)` agrupado por `(municipio, año)`. Socrata no soporta `COUNT(DISTINCT ...)` en SoQL, por lo que este valor cuenta observaciones, no estaciones únicas. Para el MVP es una aproximación aceptable.

### Valores de precipitación negativos o extremos

- Valores < 0 → `NaN` (imposible físicamente); loggear municipio y año
- Valores > 1.500 mm/año → loggear warning pero conservar

### Municipios con cobertura parcial

Si un municipio tiene datos solo para algunos años, las filas faltantes quedarán ausentes del DataFrame. D2.4 maneja los huecos al construir la serie completa.
