## Objetivo
Descargar la precipitación del dataset IDEAM `s54a-sgyg` para las estaciones de los 15 municipios del MVP, agregada por municipio y año, y retornar un DataFrame con acumulado anual, días secos y días de lluvia.

### Columnas relevantes en la API
| Columna API | Tipo | Descripción |
|-------------|------|-------------|
| `codigoestacion` | string | Código de la estación (une con D2.1) |
| `municipio` | string | Nombre del municipio (MAYÚSCULAS) |
| `fechaobservacion` | string | Fecha y hora de la observación (ISO 8601) |
| `valorobservado` | string | Valor de precipitación en mm (viene como string) |

### Consulta SoQL implementada
Se hacen **dos consultas separadas** para obtener las dos métricas necesarias:

**Consulta A — Acumulado anual y conteo de días:**
```sql
$select=municipio,
        date_trunc_y(fechaobservacion) AS año,
        sum(valorobservado) AS prec_acum_mm,
        count(*) AS n_observaciones
$where=codigoestacion IN ('...lista de códigos de D2.1...')
       AND valorobservado IS NOT NULL
$group=municipio, date_trunc_y(fechaobservacion)
$order=municipio, año
```

**Consulta B — Días secos y días de lluvia (umbral 1 mm):**
```sql
$select=municipio,
        date_trunc_y(fechaobservacion) AS año,
        count(CASE WHEN valorobservado < 1 THEN 1 END) AS prec_dias_secos,
        count(CASE WHEN valorobservado >= 1 THEN 1 END) AS prec_dias_lluvia
$where=codigoestacion IN ('...lista de códigos de D2.1...')
       AND valorobservado IS NOT NULL
$group=municipio, date_trunc_y(fechaobservacion)
$order=municipio, año
```

> **Nota sobre `date_trunc_y`**: La función de Socrata para truncar a año es `date_trunc_y(campo)`. El resultado viene como string ISO, ej. `'2015-01-01T00:00:00.000'`. Se extrae el año con `pd.to_datetime(...).dt.year`.

> **Nota sobre el filtro de estaciones**: Los códigos de estación se obtienen del DataFrame de D2.1. Si hay más de 500 estaciones, dividir en lotes de 500 para no exceder el límite de longitud de la URL.

## Lógica de transformación
```
1. Recibir df_estaciones (output de D2.1)
   Extraer lista de codigo_estacion para los 15 municipios del MVP
   Si la lista está vacía → lanzar ValueError

2. Construir filtro SoQL con los códigos de estación
   Si len(codigos) > 500 → dividir en lotes y hacer múltiples llamadas

3. Ejecutar Consulta A (acumulado + conteo):
   fetch_all(dataset_id='s54a-sgyg', select=..., where=..., group=..., order=...)
   → df_acum

4. Ejecutar Consulta B (días secos/lluvia):
   fetch_all(dataset_id='s54a-sgyg', select=..., where=..., group=..., order=...)
   → df_dias

5. Convertir columna 'año' en ambos DataFrames:
   pd.to_datetime(año_str, errors='coerce').dt.year → int

6. Convertir columnas numéricas con pd.to_numeric(..., errors='coerce'):
   prec_acum_mm, n_observaciones, prec_dias_secos, prec_dias_lluvia

7. normalize_title_case(municipio) en ambos DataFrames

8. Cruzar municipio → codigo_dane usando get_codigo() de shared/dane_codes.py

9. Filtrar: conservar solo filas donde codigo_dane esté en MVP_CODIGOS

10. Hacer merge de df_acum y df_dias por (codigo_dane, municipio, año)
    Usar how='outer' para no perder filas con datos parciales

11. Filtrar años fuera de rango: conservar solo 2007 ≤ año ≤ 2024

12. Agregar columna n_estaciones_prec:
    Para cada (municipio, año), contar cuántas estaciones distintas aportaron datos
    (requiere una tercera consulta ligera o estimación desde n_observaciones)
    Ver nota de implementación abajo.

13. Loggear warning por cada municipio del MVP sin datos de precipitación
14. Loggear resumen: registros por municipio, rango de años, % de NaN en prec_acum_mm

15. Retornar DataFrame con columnas:
    codigo_dane, municipio, año, prec_acum_mm, prec_dias_secos, prec_dias_lluvia, n_estaciones_prec
```

## Notas de implementación

### `n_estaciones_prec` — tercera consulta ligera
Para saber cuántas estaciones aportaron datos por municipio y año, se hace una consulta adicional con `count(DISTINCT codigoestacion)`:
```sql
$select=municipio,
        date_trunc_y(fechaobservacion) AS año,
        count(codigoestacion) AS n_estaciones_prec
$where=codigoestacion IN ('...')
$group=municipio, date_trunc_y(fechaobservacion)
```
> Socrata no soporta `COUNT(DISTINCT ...)` en SoQL. Se usa `count(codigoestacion)` como aproximación — cuenta observaciones, no estaciones únicas. Para el MVP (15 municipios, ~18 años) esto es aceptable. Si se necesita exactitud, se puede calcular en Python agrupando el resultado de la Consulta A antes de agregar.

### Lotes de estaciones
Si `len(codigos_estacion) > 500`, dividir en sublistas y hacer una llamada por lote. Concatenar los resultados antes de continuar con el paso 5.

### Valores de precipitación negativos o extremos
- Valores < 0 → `None` (imposible físicamente)
- Valores > 1.500 mm/año para un municipio → loggear warning pero conservar (puede ser válido en zonas muy lluviosas como Chocó; los municipios del MVP no deberían tener este problema)

### Municipios con cobertura parcial
Si un municipio tiene datos solo para algunos años, las filas faltantes quedarán ausentes del DataFrame (no se imputan). D2.4 manejará los huecos al construir la serie completa.

|-----------|---------------|
| `df_estaciones` vacío | Lanzar `ValueError("D2.2: df_estaciones vacío — ejecutar D2.1 primero")` |

| Valor de precipitación negativo | → `None`; loggear municipio y año |
| Error HTTP de Socrata | `fetch_all()` reintenta 3 veces; si falla, propagar excepción |
| DataFrame vacío tras filtros | Lanzar `ValueError("D2.2: sin registros de precipitación para el MVP")` |
