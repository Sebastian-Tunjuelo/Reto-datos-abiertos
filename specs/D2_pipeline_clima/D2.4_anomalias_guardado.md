## Agente

**Rol**: Eres un ingeniero de datos Python responsable de la capa de orquestación del pipeline climático. Tu trabajo es recibir los tres DataFrames de D2.2 y D2.3, calcular las anomalías climáticas respecto a la media histórica, unificar todo en un único DataFrame y persistirlo en disco.

**Mentalidad**: Piensa como el último guardián de calidad antes de que los datos climáticos lleguen al modelo. Las anomalías son variables críticas para el modelo predictivo — un error en su cálculo se propaga silenciosamente a todas las predicciones. Sé explícito en los logs. Si un municipio tiene huecos en alguna variable, el registro debe existir de todas formas con `None` en esa columna — no eliminar filas.

**Restricciones**:
- No re-limpiar columnas individuales — eso es responsabilidad de D2.2 y D2.3
- No eliminar filas por valores nulos — conservar todas las combinaciones (municipio, año) presentes en al menos una fuente
- No lanzar excepciones por huecos de cobertura — solo loggear
- Sí lanzar `AssertionError` si la integridad del merge falla
- El orden de columnas en el Parquet debe seguir el esquema unificado definido en el README

## Lógica de transformación

### Paso 1 — Construir el universo de combinaciones (municipio, año)
```
Crear un DataFrame base con todas las combinaciones posibles:
  municipios: MVP_CODIGOS (15 municipios)
  años: 2007–2024 (18 años)
  → 15 × 18 = 270 filas

Esto garantiza que el output tenga una fila por cada combinación,
incluso si alguna variable climática no tiene datos para ese año.
```

### Paso 2 — Merge de las tres fuentes
```
df_base
  LEFT JOIN df_prec  ON (codigo_dane, año)
  LEFT JOIN df_temp  ON (codigo_dane, año)
  LEFT JOIN df_hum   ON (codigo_dane, año)

Usar how='left' para conservar todas las combinaciones del universo.
Resolver conflictos de columna 'municipio': conservar la del df_base
(derivada de DANE_TO_NAME — es la fuente de verdad).
```

### Paso 3 — Rellenar `n_estaciones_prec` y `n_estaciones_temp`
```
Donde n_estaciones_prec es NaN (municipio/año sin datos de precipitación) → 0
Donde n_estaciones_temp es NaN (municipio/año sin datos de temperatura) → 0
Convertir a int.
```

### Paso 4 — Calcular anomalías
```
Período de referencia histórica: años donde año <= TRAIN_HASTA (2021)
  (importar TRAIN_HASTA de shared/config.py)

Para cada municipio:
  media_hist_prec[municipio] = prec_acum_mm[año <= 2021].mean()
  media_hist_temp[municipio] = temp_media_c[año <= 2021].mean()

anomalia_prec = (prec_acum_mm - media_hist_prec) / media_hist_prec
  → fracción: 0.0 = normal, -0.3 = 30% menos lluvia, +0.2 = 20% más lluvia
  → Si media_hist_prec == 0 o es NaN → anomalia_prec = None

anomalia_temp = temp_media_c - media_hist_temp
  → grados absolutos: 0.0 = normal, +1.5 = 1.5°C más caliente
  → Si media_hist_temp es NaN → anomalia_temp = None

Loggear por municipio: media histórica de precipitación y temperatura usada
```

### Paso 5 — Validaciones de integridad
```
Verificar que el DataFrame tiene exactamente 270 filas (15 × 18)
  Si no → loggear warning con el detalle (no lanzar excepción)

Verificar que no hay duplicados por (codigo_dane, año)
  Si hay → lanzar AssertionError

Verificar que todos los MVP_CODIGOS están presentes
  Si falta alguno → lanzar AssertionError
```

### Paso 6 — Ordenar y guardar
```
Ordenar por (codigo_dane, año)
Seleccionar columnas en el orden exacto del esquema
Guardar: output_dir / 'clima_agregado.parquet'
  df.to_parquet(path, index=False, engine='pyarrow')
```

### Paso 7 — Resumen de ejecución
```
Imprimir con logging.info:
  - Total de filas
  - Cobertura por variable: % de filas con datos (no NaN)
  - Municipios con cobertura completa (todas las variables, todos los años)
  - Municipios con cobertura parcial (alguna variable o año faltante)
  - Media histórica de precipitación y temperatura por municipio
```

## Notas de implementación

### `calcular_anomalias` como función separada
La lógica del paso 4 se implementa en `modules/climate/aggregation.py → calcular_anomalias()`. Esto permite testearla de forma aislada y reutilizarla si se agregan nuevas variables en el futuro.

```python

    año_corte: int,  # TRAIN_HASTA = 2021

    a la media histórica del período hasta año_corte.
    Retorna df con columnas anomalia_prec y anomalia_temp añadidas.
    """
```

### Municipios con menos de 3 años de datos históricos
Si un municipio tiene menos de 3 años con datos de precipitación o temperatura en el período histórico (2007–2021), la media histórica es poco confiable. Loggear warning: `"[D2.4] {municipio}: solo {n} años de datos históricos para {variable} — anomalía poco confiable"`. Calcular de todas formas.

### Columna `municipio` en el merge
Los tres DataFrames de entrada tienen columna `municipio`. Al hacer el merge, pandas añade sufijos (`_x`, `_y`). Estrategia: antes del merge, renombrar `municipio` en df_prec, df_temp y df_hum a `municipio_prec`, `municipio_temp`, `municipio_hum`, hacer el merge solo por `codigo_dane` y `año`, y luego derivar `municipio` desde `DANE_TO_NAME[codigo_dane]` — que es la fuente de verdad.
