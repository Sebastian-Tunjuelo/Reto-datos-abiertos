## Agente

**Rol**: Eres un ingeniero de datos Python especializado en ingesta de APIs REST con datos geoespaciales. Tu trabajo es descargar la frontera agrícola nacional de UPRA para los 15 municipios del MVP, agregar el área por tipo de frontera y retornar un DataFrame con los porcentajes de área condicionada y no condicionada.

**Mentalidad**: La frontera agrícola delimita dónde se puede expandir la producción agropecuaria. Para el modelo predictivo, el porcentaje de área condicionada (`pct_condicionada`) es una variable de restricción: un municipio con alta proporción de área no condicionada tiene menos espacio para crecer. Este dato es estático — no cambia por año ni por cultivo. El output es una tabla de lookup por `codigo_dane` que D5 une a todos los cultivos.

**Restricciones**:
- No inventar lógica de negocio que no esté en esta spec
- No descargar geometrías — siempre `$group` + `sum(area_ha)` en SoQL
- No usar `sodapy` — usar `shared/socrata_client.py` directamente
- No usar `inplace=True` en operaciones Pandas
- No hardcodear los códigos DANE — usar `shared/dane_codes.py`
- El output no tiene columna `cultivo` — la frontera aplica igual a todos los cultivos

## Objetivo
Descargar la frontera agrícola UPRA para los 15 municipios del MVP, calcular el porcentaje de área condicionada y no condicionada por municipio, y retornar un DataFrame con el esquema de frontera del proyecto.

## Lógica de transformación
```
1. Llamar fetch_all(dataset_id='fyc7-sbtz', select=..., where=..., group=...)
2. Convertir lista de dicts → pd.DataFrame
   Si DataFrame vacío → lanzar ValueError("Frontera agrícola: sin registros para el MVP")

3. Renombrar columnas: cod_dane_m → codigo_dane, departamen → departamento,
   area_total → area_ha_raw

4. normalize_dane_code(codigo_dane) → str 5 dígitos
5. Filtrar: conservar solo filas donde codigo_dane esté en MVP_CODIGOS
6. normalize_title_case(municipio), normalize_title_case(departamento)
7. area_ha_raw = pd.to_numeric(area_ha_raw, errors='coerce')

8. Normalizar tipo de frontera:
   tipo_norm = _normalizar_frontera(tipo_front)
   → 'Condicionada' | 'No_condicionada' | None
   Filas con tipo None → loggear y descartar

9. Pivot: una fila por municipio, columnas area_condicionada_ha, area_no_condicionada_ha
   df.pivot_table(index=['codigo_dane','municipio','departamento'],
                  columns='tipo_norm', values='area_ha_raw',
                  aggfunc='sum').reset_index()
   Rellenar NaN con 0.0

10. Calcular area_total_ha = area_condicionada_ha + area_no_condicionada_ha
    Si area_total_ha == 0 → loggear warning, dejar pct_* como None

11. Calcular pct_condicionada = area_condicionada_ha / area_total_ha
    Calcular pct_no_condicionada = area_no_condicionada_ha / area_total_ha
    Redondear a 4 decimales

12. Loggear warning por cada municipio del MVP sin datos
13. Retornar DataFrame con columnas en orden del esquema de salida
```

---

## Función privada `_normalizar_frontera`
```python
_FRONTERA_MAP = {
    'condicionada': 'Condicionada',
    'c': 'Condicionada',
    'no condicionada': 'No_condicionada',
    'no_condicionada': 'No_condicionada',
    'nc': 'No_condicionada',
}

def _normalizar_frontera(valor: str | None) -> str | None:
    if valor is None:
        return None
    return _FRONTERA_MAP.get(valor.strip().lower(), None)
```

## Notas de implementación

### Sin columna `cultivo`
La frontera agrícola no depende del cultivo — aplica igual a café, cacao y maíz. D5 hará un join por `codigo_dane` solamente y replicará el valor para los 3 cultivos.

### Interpretación de `pct_condicionada`
Un valor alto de `pct_condicionada` indica que el municipio tiene mucha área dentro de la frontera agrícola (apta para producción). Un valor bajo indica restricciones de uso del suelo. En el modelo, esta variable actúa como moderador del potencial productivo.
