## Objetivo
Descargar la aptitud agrícola de UPRA para café y cacao en los 15 municipios del MVP, calcular los porcentajes de área por categoría y retornar un DataFrame por cultivo con el esquema unificado.

|-------------|------|-------------|
| `cod_dane_m` | string | Código DANE municipio (5 dígitos, bien formateado) |
| `municipio` | string | Nombre en MAYÚSCULAS |
| `departamen` | string | Nombre del departamento en MAYÚSCULAS (truncado a 10 chars) |

### Query SoQL implementada (igual para ambos datasets, cambia el dataset_id)
```
$select=cod_dane_m,municipio,departamen,aptitud,sum(area_ha) AS area_total
$where=cod_dane_m IN ('73001','73168','41001','41298','41551','68689','68615',
  '05036','05030','17541','17524','50001','19256','19418','20001')
$group=cod_dane_m,municipio,departamen,aptitud
$order=cod_dane_m,aptitud
```
> El `$group` es obligatorio. Sin él, la API retorna una fila por polígono (geometría incluida), lo que puede ser decenas de miles de registros y rompe la paginación.

## Mapeo de columnas → esquema unificado
| Columna API / derivada | Columna unificada | Transformación |
|------------------------|-------------------|----------------|
| `cod_dane_m` | `codigo_dane` | `normalize_dane_code()` |
| `municipio` | `municipio` | `normalize_title_case()` |
| `departamen` | `departamento` | `normalize_title_case()` (campo truncado — ver nota) |
| `aptitud` | *(pivot)* | normalizar → pivot → columnas `area_*_ha` |
| `area_total` | `area_*_ha` | `pd.to_numeric(..., errors='coerce')` |
| *(agregar)* | `cultivo` | `'Café'` o `'Cacao'` según la función |
| *(calcular)* | `area_total_ha` | suma de las 4 categorías |
| *(calcular)* | `pct_alta`, `pct_media`, `pct_baja`, `pct_exclusion` | `area_*_ha / area_total_ha` |

## Lógica de transformación
```
1. Llamar fetch_all(dataset_id=<id>, select=<select>, where=<where>, group=<group>)
   Resultado: lista de dicts con una fila por (municipio, aptitud)

2. Convertir lista de dicts → pd.DataFrame
   Si DataFrame vacío → lanzar ValueError("Aptitud {cultivo}: sin registros para el MVP")

3. Renombrar columnas: cod_dane_m → codigo_dane, departamen → departamento,
   area_total → area_ha_raw

4. normalize_dane_code(codigo_dane) → str 5 dígitos
5. Filtrar: conservar solo filas donde codigo_dane esté en MVP_CODIGOS
6. normalize_title_case(municipio), normalize_title_case(departamento)
7. area_ha_raw = pd.to_numeric(area_ha_raw, errors='coerce')

8. Normalizar categoría de aptitud:
   aptitud_norm = _normalizar_aptitud(aptitud)
   → 'Alta' | 'Media' | 'Baja' | 'Exclusion' | None
   Filas con aptitud None → loggear y descartar

9. Pivot: una fila por municipio, columnas area_alta_ha, area_media_ha,
   area_baja_ha, area_exclusion_ha
   Usar: df.pivot_table(index=['codigo_dane','municipio','departamento'],
                        columns='aptitud_norm', values='area_ha_raw',
                        aggfunc='sum').reset_index()
   Rellenar NaN con 0.0 para categorías sin área en ese municipio

10. Calcular area_total_ha = area_alta_ha + area_media_ha + area_baja_ha + area_exclusion_ha
    Si area_total_ha == 0 → loggear warning y dejar pct_* como None

11. Calcular pct_alta = area_alta_ha / area_total_ha (etc. para las 4 categorías)
    Redondear a 4 decimales

12. Agregar columna cultivo = 'Café' o 'Cacao'

13. Loggear warning por cada municipio del MVP sin datos
14. Retornar DataFrame con columnas en orden del esquema unificado
```

## Función privada `_normalizar_aptitud`
```python
_APTITUD_MAP = {
    'alta': 'Alta',
    'a': 'Alta',
    'media': 'Media',
    'm': 'Media',
    'moderada': 'Media',
    'baja': 'Baja',
    'b': 'Baja',
    'exclusión': 'Exclusion',
    'exclusion': 'Exclusion',
    'no apta': 'Exclusion',
    'e': 'Exclusion',
}

def _normalizar_aptitud(valor: str | None) -> str | None:
    if valor is None:
        return None
    return _APTITUD_MAP.get(valor.strip().lower(), None)
```
> Esta función es privada y compartida entre D3.1, D3.2 y D3.3. Definirla una sola vez en `modules/territorial/ingestion.py`.

## Nota sobre `departamen`
El campo `departamen` en la API UPRA está truncado a ~10 caracteres (ej. `'ANTIOQUIA'` → `'ANTIOQUIA'`, `'NORTE DE S'` en lugar de `'NORTE DE SANTANDER'`). Para los 15 municipios del MVP se puede derivar el departamento completo desde `DANE_TO_DEPT[codigo_dane]` si el valor truncado es ambiguo. En la práctica, para el MVP los nombres no son ambiguos, pero se aplica `normalize_title_case()` de todas formas.

## Notas de implementación

### Por qué pivot y no groupby directo
La API retorna una fila por `(municipio, aptitud)` tras el `$group`. Para el esquema de salida (una fila por municipio con columnas por categoría) se necesita un pivot. `pivot_table` con `aggfunc='sum'` es más robusto que `pivot` porque maneja duplicados inesperados sin lanzar excepción.

### Municipios sin alguna categoría de aptitud
Si un municipio no tiene área de aptitud `'Alta'` (por ejemplo), el pivot genera `NaN` para `area_alta_ha`. Se rellena con `0.0` antes de calcular `area_total_ha` y los porcentajes. Esto es correcto: ausencia de área Alta significa 0 ha, no dato faltante.
