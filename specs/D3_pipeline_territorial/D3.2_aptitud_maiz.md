## Agente

**Rol**: Eres un ingeniero de datos Python especializado en ingesta y consolidación de fuentes heterogéneas. Tu trabajo es descargar los 3 datasets de aptitud maíz de UPRA, consolidarlos en un único DataFrame y retornar la mejor estimación de aptitud por municipio.

**Mentalidad**: Maíz es el cultivo más complejo en UPRA porque tiene 3 datasets distintos: maíz tradicional, maíz tecnificado 1er semestre y maíz tecnificado 2do semestre. No todos los municipios aparecen en los 3 datasets — maíz tradicional solo cubre 123 municipios en Colombia. La estrategia de consolidación es tomar el **máximo de área por categoría** entre los 3 datasets para cada municipio: si un municipio tiene aptitud Alta en maíz tecnificado pero no aparece en maíz tradicional, se usa el tecnificado. El consumidor (D5) no necesita saber de qué dataset vino — solo necesita la mejor estimación disponible.

**Restricciones**:
- No inventar lógica de negocio que no esté en esta spec
- No descargar geometrías — siempre `$group` + `sum(area_ha)` en SoQL

- No usar `inplace=True` en operaciones Pandas
- No hardcodear los códigos DANE — usar `shared/dane_codes.py`
- Si un municipio no aparece en ninguno de los 3 datasets → loggear warning, no fallar

## Lógica de consolidación
```
1. Para cada uno de los 3 datasets:
   a. Llamar fetch_all(dataset_id=<id>, select=..., where=..., group=...)
   b. Si retorna vacío → loggear warning "[D3.2] Dataset {id}: sin datos para el MVP"
      (no lanzar excepción — puede ser que maíz tradicional no cubra estos municipios)
   c. Convertir a DataFrame, renombrar columnas, normalize_dane_code, normalize_title_case
   d. _normalizar_aptitud(aptitud) → 'Alta'|'Media'|'Baja'|'Exclusion'|None
   e. Descartar filas con aptitud None; loggear valores inesperados
   f. area_ha = pd.to_numeric(area_total, errors='coerce')
   g. Agregar columna fuente = 'tradicional' | 'tec_1sem' | 'tec_2sem'

2. Concatenar los 3 DataFrames: df_todos = pd.concat([df1, df2, df3], ignore_index=True)
   Si df_todos está vacío → lanzar ValueError("Aptitud maíz: sin registros para el MVP")

3. Consolidar por (codigo_dane, municipio, departamento, aptitud_norm):
   Tomar el MÁXIMO de area_ha entre los 3 datasets para cada combinación.
   Razón: si un municipio tiene más área de aptitud Alta en tecnificado que en tradicional,
   usamos el mayor — representa la mejor estimación disponible.
   df_consolidado = df_todos.groupby(
       ['codigo_dane', 'municipio', 'departamento', 'aptitud_norm'],
       as_index=False
   )['area_ha'].max()

4. Pivot: una fila por municipio, columnas area_alta_ha, area_media_ha,
   area_baja_ha, area_exclusion_ha
   (mismo procedimiento que D3.1 — ver función privada _pivot_aptitud)

5. Calcular area_total_ha y pct_* (mismo procedimiento que D3.1)

6. Agregar columna cultivo = 'Maíz'

7. Loggear resumen: cuántos municipios vinieron de cada dataset
8. Loggear warning por cada municipio del MVP sin datos en ningún dataset
9. Retornar DataFrame con columnas en orden del esquema unificado
```

## Estrategia de consolidación — justificación

| Opción | Descripción | Problema |
|--------|-------------|---------|
| Solo maíz tradicional | Más conservador | Solo 123 municipios — puede dejar municipios sin datos |
| Solo maíz tecnificado | Más cobertura | No refleja aptitud para producción tradicional |
| **Máximo entre los 3** ✅ | Mejor estimación disponible | Puede sobreestimar ligeramente la aptitud |
| Promedio entre los 3 | Más conservador | Penaliza municipios que solo aparecen en 1 dataset |

Se elige **máximo** porque el objetivo del modelo es identificar el potencial productivo del municipio. Si el municipio tiene aptitud Alta para maíz tecnificado, esa información es relevante aunque no tenga datos de maíz tradicional.

```python
def download_aptitud_maiz(
    codigos_dane: list[str] | None = None,
) -> pd.DataFrame:
    """
    Descarga y consolida aptitud agrícola UPRA para maíz desde 3 datasets.

    Descarga frjn-92um (tradicional), a5yc-uszt (tec. 1er sem) y
    tzga-4zse (tec. 2do sem). Consolida tomando el máximo de área
    por categoría entre los 3 datasets para cada municipio.

    Args:
        codigos_dane: Lista de códigos DANE a filtrar.

                  area_alta_ha, area_media_ha, area_baja_ha, area_exclusion_ha,
                  area_total_ha, pct_alta, pct_media, pct_baja, pct_exclusion.

    Raises:
        ValueError: Si los 3 datasets están vacíos tras aplicar filtros.
        requests.RequestException: Si alguna descarga falla tras reintentos.
    """
```
