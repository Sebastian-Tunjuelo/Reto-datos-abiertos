## Agente

**Rol**: Eres un ingeniero de QA de modelos. Tu tarea es validar de forma independiente que los artefactos de M3 existen, se pueden cargar y respetan el contrato esperado.

**Mentalidad**: Piensa como auditor. No entrenes ni recalcules nada; solo comprueba que los artefactos finales son consistentes y que el pipeline dejó salidas utilizables.

**Restricciones**:

- No modificar archivos ni regenerar artefactos
- No llamar al pipeline de entrenamiento
- Falla rápido con mensajes claros y accionables

## Objetivo

Escribir un script automatizado `validate_m3.py` para verificar que el pipeline de clasificación completo terminó bien y que los artefactos de salida son válidos.

## Pruebas de ejecución

1. Verificar la existencia de los 3 `.pkl`, los 3 `*_meta.json` y `models/m3_classification_metrics.json`.
2. Cargar el JSON de métricas y comprobar que tiene 9 registros, uno por cultivo y split.
3. Validar que cada registro contiene `cultivo`, `split`, `n`, `accuracy`, `precision`, `recall`, `f1_score`, `auc_roc`, `best_iteration`.
4. Comprobar que `split` solo toma valores `train`, `val` y `test`, y que `cultivo` pertenece a `Café`, `Cacao` o `Maíz`.
5. Cargar cada modelo con `joblib.load()` y confirmar que expone `predict_proba`.
6. Cargar cada meta.json y comprobar que contiene `feature_cols` como lista no vacía y `best_iteration` como entero positivo.
7. Revisar que `auc_roc` esté en rango [0.0, 1.0] y que `n` sea mayor que 0 en todos los registros.

---

## Salida esperada

Salida estándar por consola indicando que M3 está listo para integrarse al resto del sistema. El script debe devolver `Exit Code 0` cuando todo pase.

---

## Manejo de errores

| Situación                                        | Comportamiento                                           |
| ------------------------------------------------ | -------------------------------------------------------- |
| Falta un archivo requerido                       | `AssertionError` o `SystemExit(1)` con mensaje explícito |
| JSON inválido o incompleto                       | `AssertionError` con el campo faltante                   |
| Modelo sin `predict_proba`                       | `AssertionError`                                         |
| Métricas fuera de rango o con esquema incorrecto | `AssertionError`                                         |

- Dado un archivo de métricas bien formado
- Cuando se inspecciona su contenido

- Dado un modelo serializado

- Entonces sigue exponiendo `predict_proba`
