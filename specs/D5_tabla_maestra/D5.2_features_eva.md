## Agente

**Rol**: Eres un ingeniero de ML Python especializado en feature engineering para series de tiempo agrícolas. Tu trabajo es construir las variables rezagadas y derivadas a partir de la serie histórica de rendimiento y área sembrada de EVA, respetando estrictamente la regla de no fuga de información temporal.

**Mentalidad**: Piensa en términos de lo que el modelo puede saber en el momento de la predicción. Para predecir el rendimiento del año `t`, el modelo solo puede usar datos de años `t-1`, `t-2`, `t-3`. Cualquier dato del año `t` (excepto variables exógenas como clima o agroinsumos que se conocen al inicio del ciclo) es fuga. Sé explícito en los nombres de columnas — `rendimiento_t1` significa "rendimiento del año t-1", no "rendimiento con lag 1 en el índice del DataFrame".

**Restricciones**:
- No usar `rendimiento`, `area_cosechada` ni `produccion` del mismo año como features
- No usar `shift()` sin ordenar primero por `(codigo_dane, cultivo, año)` — los lags serían incorrectos
- No usar `inplace=True` en Pandas
- No imputar NaN en features rezagadas — son válidos para años sin historial suficiente
- Si algo no está especificado, preguntar antes de asumir

# Tendencia lineal de los últimos 3 años
# Pendiente de regresión lineal sobre [t-3, t-2, t-1]

## Notas de implementación

### Agregación de periodos A/B
Algunos cultivos transitorios (Maíz) tienen registros `'2015A'` y `'2015B'` en el mismo año. Para el modelo se necesita un único valor anual. La lógica de agregación:

```python
def agregar_anual(df_eva: pd.DataFrame) -> pd.DataFrame:
    """Agrega EVA a nivel (codigo_dane, cultivo, año)."""
    # Si ya hay una sola fila por (codigo_dane, cultivo, año), no hace nada
    llave = ['codigo_dane', 'cultivo', 'año']
    if not df_eva.duplicated(llave).any():
        return df_eva[llave + ['municipio', 'departamento', 'rendimiento', 'area_sembrada']]

    def prom_ponderado(g):
        # Ponderar por area_cosechada si está disponible y no es todo NaN
        if 'area_cosechada' in g.columns and g['area_cosechada'].notna().any():
            pesos = g['area_cosechada'].fillna(0)
            if pesos.sum() > 0:
                rend = (g['rendimiento'] * pesos).sum() / pesos.sum()
            else:
                rend = g['rendimiento'].mean()
        else:
            rend = g['rendimiento'].mean()
        return pd.Series({
            'rendimiento': rend,
            'area_sembrada': g['area_sembrada'].sum(),
            'municipio': g['municipio'].iloc[0],
            'departamento': g['departamento'].iloc[0],
        })

    return df_eva.groupby(llave).apply(prom_ponderado).reset_index()
```

### Tendencia lineal de 3 años
No existe una función nativa de pandas para la pendiente de una ventana deslizante. Usar `apply` con `numpy.polyfit`:

```python
import numpy as np

def pendiente_3a(series: pd.Series) -> pd.Series:
    """Calcula la pendiente lineal de los últimos 3 años (shift=1)."""
    shifted = series.shift(1)
    def _slope(window):
        vals = window.dropna()
        if len(vals) < 2:
            return np.nan
        x = np.arange(len(vals))
        return np.polyfit(x, vals, 1)[0]
    return shifted.rolling(window=3, min_periods=2).apply(_slope, raw=False)
```

El costo de `apply` es aceptable dado el volumen (~15 municipios × 3 cultivos × 18 años = ~810 filas).

## Manejo de errores
| Situación | Comportamiento |
|-----------|---------------|
| `df_eva` vacío | `ValueError("D5.2: df_eva está vacío")` |
