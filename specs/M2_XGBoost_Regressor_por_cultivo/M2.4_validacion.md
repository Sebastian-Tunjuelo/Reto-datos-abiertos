---

## Agente

**Rol**: Eres un ingeniero de ML encargado de verificar que los artefactos de M2
esten completos y consistentes.

**Mentalidad**: Si falta un artefacto o las metricas no son validas, falla con un
mensaje claro y accionable.

---

## Objetivo

Validar la existencia de los modelos, metadatos y metricas, y realizar chequeos
basicos de consistencia.

---

## Inputs

- `models/xgb_regressor_cafe.pkl`
- `models/xgb_regressor_cacao.pkl`
- `models/xgb_regressor_maiz.pkl`
- `models/xgb_regressor_*_meta.json`
- `models/m2_regression_metrics.json`

---

## Logica de validacion

```
1. Verificar que existen los 3 modelos .pkl
2. Verificar que existen los 3 archivos meta.json
3. Verificar que existe m2_regression_metrics.json
4. Cargar metrics y comprobar:
   - 9 registros (3 cultivos x 3 splits)
   - campos requeridos: cultivo, split, n, mae, rmse, r2
   - n > 0, mae/rmse >= 0
   - r2 en rango [-1.0, 1.0] o None
5. Cargar cada meta.json y comprobar:
   - contiene feature_cols (lista no vacia)
   - contiene best_iteration (int >= 1)
```

---

## Output

- Script `validate_m2.py` que imprime resumen y retorna exit code 0 si todo ok

---

## Manejo de errores

| Situacion                        | Comportamiento                          |
| -------------------------------- | --------------------------------------- |
| Falta algun archivo requerido    | `SystemExit(1)` con mensaje descriptivo |
| JSON invalido o campos faltantes | `SystemExit(1)`                         |
| Metricas fuera de rango          | `SystemExit(1)`                         |

---

## Criterios de aceptacion

- [x] El script falla si falta cualquier modelo o metricas
- [x] El script valida 9 registros de metricas
- [x] El script valida que `feature_cols` exista en cada meta.json
- [x] Exit code 0 cuando todo esta completo
