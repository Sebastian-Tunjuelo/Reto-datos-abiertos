## Agente

**Rol**: Eres un ingeniero de ML responsable del entrenamiento de modelos de regresion.

**Mentalidad**: Prioriza split temporal, reproducibilidad y trazabilidad de artefactos.

**Restricciones**:

- No entrenar un modelo global (solo por cultivo)
- No usar split aleatorio
- No usar data de test en training

---

## Objetivo

Entrenar un modelo XGBRegressor por cultivo usando split temporal estricto y
guardar los modelos en `models/`.

---

## Inputs

- DataFrame validado (output de M2.1)
- `feature_cols`
- `target_col = "target_rendimiento"`
- Constantes de `shared/config.py`: `CULTIVOS_MVP`, `TRAIN_HASTA`, `VAL_AÑO`, `TEST_DESDE`, `MODELS_DIR`

---

## Parametros base del modelo

```python
XGBRegressor(
    objective="reg:squarederror",
    n_estimators=500,
    learning_rate=0.05,
    max_depth=6,
    subsample=0.8,
    colsample_bytree=0.8,
    reg_alpha=0.0,
    reg_lambda=1.0,
    random_state=42,
    n_jobs=-1,
)
```

Entrenar con:

- `eval_metric="rmse"`
- `early_stopping_rounds=30`
- `eval_set=[(X_train, y_train), (X_val, y_val)]`

---

## Logica de entrenamiento

```
Para cada cultivo en CULTIVOS_MVP:
1. Filtrar df por cultivo
2. Split temporal:
   - train: año <= TRAIN_HASTA
   - val:   año == VAL_AÑO
   - test:  año >= TEST_DESDE
3. Validar que train/val/test tengan filas
   - Si alguno esta vacio -> ValueError("M2.2: split vacio para {cultivo}")
4. Construir X/y con feature_cols y target_col
5. Entrenar XGBRegressor con early stopping
6. Guardar modelo en:
   models/xgb_regressor_{slug}.pkl
7. Guardar meta en:
   models/xgb_regressor_{slug}_meta.json
   (feature_cols, target_col, years_split, best_iteration)
```

`slug` se define como: `cafe`, `cacao`, `maiz`.

---

## Output

- 3 modelos `.pkl` (uno por cultivo)
- 3 archivos `.json` de metadatos
