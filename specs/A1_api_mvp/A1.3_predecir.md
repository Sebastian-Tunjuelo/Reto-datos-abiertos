# A1.3 - Predicción de rendimiento y riesgo

## Metadatos

| Campo         | Valor                                                                                                                                                                                                                                                  |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| ID            | A1.3                                                                                                                                                                                                                                                   |
| Módulo o área | `orchestrator/main.py` → `POST /predecir`                                                                                                                                                                                                              |
| Depende de    | A1.1, A1.2, `data/feature_matrix.parquet`, `models/xgb_regressor_*.pkl`, `models/xgb_regressor_*_meta.json`, `models/xgb_classifier_*.pkl`, `models/xgb_classifier_*_meta.json`, `shared/config.py`, `shared/dane_codes.py`, `shared/normalization.py` |

## Agente

- role: Ingeniero backend / ML integrado.
- goal: Convertir una petición municipio + cultivo + año en una predicción estable de rendimiento y riesgo alto.
- backstory: Este endpoint orquesta los artefactos de M2 y M3 sin exponer detalles internos del entrenamiento.

## Objetivo

- Devolver una predicción por municipio y cultivo, usando el último contexto histórico disponible como base de features y los modelos específicos por cultivo.

## Alcance

### Incluye

- `POST /predecir` como endpoint principal de la API.
- Normalización de municipio y cultivo antes de cualquier lookup.
- Selección de la última fila histórica disponible para la combinación municipio + cultivo.
- Carga del regressor y classifier correctos según el cultivo.
- Construcción del vector de features en el orden del metadato del modelo.
- Cálculo de `rendimiento_esperado`, `prob_riesgo_alto` y `etiqueta_riesgo`.

### No incluye

- SHAP.
- Escenarios contrafactuales.
- Chat o RAG.
- Persistencia de predicciones en PostgreSQL.

## Entradas

- Request JSON con `municipio`, `cultivo` y `año`.
- `data/feature_matrix.parquet`.
- Modelos y metadatos de M2 y M3.

## Inputs esperados

| Campo       | Tipo  | Regla                                                                                                |
| ----------- | ----- | ---------------------------------------------------------------------------------------------------- |
| `municipio` | `str` | Nombre display o código DANE del MVP.                                                                |
| `cultivo`   | `str` | Debe resolverse a `Café`, `Cacao` o `Maíz`.                                                          |
| `año`       | `int` | Año objetivo solicitado; debe ser posterior al último año disponible para la combinación consultada. |

## Lógica / Proceso sugerido

1. Resolver el municipio y el cultivo con las mismas reglas canónicas de A1.1 y A1.2.
2. Validar que el año sea entero y que represente una solicitud futura, no una consulta histórica.
3. Cargar `data/feature_matrix.parquet` y ubicar la última fila histórica disponible para el par municipio + cultivo.
4. Validar que exista el modelo del cultivo solicitado y que su metadato describa el orden esperado de features.
5. Construir la fila de inferencia respetando el orden del metadato y sin introducir columnas prohibidas.
6. Ejecutar el regressor para obtener `rendimiento_esperado`.
7. Ejecutar el classifier para obtener la probabilidad de clase alta; usar la clase `2` cuando el modelo sea multiclase y la clase positiva cuando sea binario.
8. Derivar `etiqueta_riesgo` con `UMBRAL_RIESGO_MEDIO` y `UMBRAL_RIESGO_ALTO`.
9. Redondear la salida y devolver un JSON estable y reproducible.

## Interfaz o contrato

### `POST /predecir`

Request `PrediccionRequest`:

| Campo       | Tipo  | Nullable | Regla                        |
| ----------- | ----- | -------- | ---------------------------- |
| `municipio` | `str` | No       | Nombre o código DANE.        |
| `cultivo`   | `str` | No       | `Café` \| `Cacao` \| `Maíz`. |
| `año`       | `int` | No       | Año objetivo de planeación.  |

Response `PrediccionResponse`:

| Campo                  | Tipo    | Nullable | Regla                            |
| ---------------------- | ------- | -------- | -------------------------------- |
| `codigo_dane`          | `str`   | No       | 5 dígitos.                       |
| `municipio`            | `str`   | No       | Title Case.                      |
| `departamento`         | `str`   | No       | Title Case.                      |
| `cultivo`              | `str`   | No       | `Café` \| `Cacao` \| `Maíz`.     |
| `año`                  | `int`   | No       | Echo del año solicitado.         |
| `rendimiento_esperado` | `float` | No       | t/ha estimadas por el regressor. |
| `prob_riesgo_alto`     | `float` | No       | Fracción en rango `[0, 1]`.      |
| `etiqueta_riesgo`      | `str`   | No       | `Bajo`, `Medio` o `Alto`.        |

Ejemplo de respuesta:

```json
{
  "codigo_dane": "73168",
  "municipio": "Chaparral",
  "departamento": "Tolima",
  "cultivo": "Café",
  "año": 2025,
  "rendimiento_esperado": 0.82,
  "prob_riesgo_alto": 0.71,
  "etiqueta_riesgo": "Alto"
}
```

## Manejo de errores

| Situación                           | HTTP | Mensaje sugerido                                                     |
| ----------------------------------- | ---- | -------------------------------------------------------------------- |
| Municipio no reconocible            | 404  | `Municipio no encontrado en el MVP`                                  |
| Cultivo inválido                    | 422  | `Cultivo inválido. Valores permitidos: Café, Cacao, Maíz`            |
| Año no entero o no futuro           | 422  | `Año objetivo inválido para predicción`                              |
| No existe una fila histórica base   | 404  | `No hay datos históricos suficientes para la combinación solicitada` |
| Falta un modelo o metadato          | 503  | `Artefactos de predicción no disponibles`                            |
| La tabla maestra no está disponible | 503  | `feature_matrix.parquet no disponible`                               |
| Error inesperado de inferencia      | 500  | `Error interno en la API de predicción`                              |

## Salida esperada

- Una respuesta con el código DANE, municipio, departamento y cultivo normalizados.
- `rendimiento_esperado` en t/ha.
- `prob_riesgo_alto` en rango `[0, 1]`.
- `etiqueta_riesgo` derivada de la probabilidad y los umbrales del proyecto.

## Caso feliz

- Dado `municipio = "Chaparral"`, `cultivo = "Café"` y `año = 2025`.
- Cuando el frontend invoca `POST /predecir`.
- Entonces la API devuelve una predicción consistente, con los identificadores normalizados y el semáforo derivado de la probabilidad.

## Validación

- Dado un municipio fuera del MVP.
- Cuando se invoca el endpoint.
- Entonces la API responde con 404.
- Dado un cultivo distinto de `Café`, `Cacao` o `Maíz`.
- Cuando se invoca el endpoint.
- Entonces la API responde con 422.
- Dado un año histórico o no válido.
- Cuando se invoca el endpoint.
- Entonces la API responde con 422.

## Criterios de aceptación

- El modelo usado corresponde al cultivo solicitado.
- La salida respeta el orden de features definido por el metadato del modelo.
- `prob_riesgo_alto` siempre queda entre 0 y 1.
- `etiqueta_riesgo` no se calcula con reglas ad hoc fuera de `shared/config.py`.
- La API no inventa features futuras ni persiste resultados.

## Restricciones técnicas

- No usar SHAP dentro de este endpoint.
- No usar escenarios ni chat.
- No usar datos de un cultivo diferente para completar la predicción.
- No alterar la base de datos.
