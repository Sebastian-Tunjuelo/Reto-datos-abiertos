# A1.2 - Cultivos por municipio

## Metadatos

| Campo         | Valor                                                                                                      |
| ------------- | ---------------------------------------------------------------------------------------------------------- |
| ID            | A1.2                                                                                                       |
| Módulo o área | `orchestrator/main.py` → `GET /cultivos/{municipio}`                                                       |
| Depende de    | A1.1, `data/feature_matrix.parquet`, `shared/dane_codes.py`, `shared/normalization.py`, `shared/config.py` |
| Produce para  | A1.3, F2, F3                                                                                               |
| Estado        | 📝 Borrador                                                                                                |

## Agente

- role: Ingeniero backend FastAPI.
- goal: Resolver un municipio del MVP y devolver únicamente los cultivos que tienen cobertura real para ese municipio.
- backstory: Esta capa alimenta el selector de cultivos y evita que el usuario elija combinaciones inválidas.

## Objetivo

- Dado un municipio del MVP, devolver el subconjunto de cultivos del MVP presentes en la base histórica del proyecto.

## Alcance

### Incluye

- `GET /cultivos/{municipio}` como endpoint read-only.
- Aceptar el municipio como nombre display o como código DANE.
- Resolver el municipio a un código DANE canónico antes de consultar datos.
- Consultar `data/feature_matrix.parquet` para inferir la cobertura real por cultivo.
- Intersectar el resultado con `CULTIVOS_MVP` y devolverlo en orden canónico.

### No incluye

- Predicción.
- Explicabilidad.
- Escenarios.
- Búsqueda fuzzy o autocompletado.

## Entradas

- `data/feature_matrix.parquet`
- `MVP_CODIGOS`
- `DANE_TO_NAME`
- `DANE_TO_DEPT`
- `CULTIVOS_MVP`

## Inputs esperados

| Campo       | Tipo  | Regla                                               |
| ----------- | ----- | --------------------------------------------------- |
| `municipio` | `str` | Puede venir como nombre display o como código DANE. |

## Lógica / Proceso sugerido

1. Normalizar el valor recibido.
2. Si el valor parece código, usar `normalize_dane_code()` y validar que pertenezca al MVP.
3. Si el valor parece nombre, resolverlo con `get_codigo()`.
4. Si no se puede resolver, devolver 404.
5. Cargar `data/feature_matrix.parquet` y verificar que tenga `codigo_dane`, `municipio`, `departamento` y `cultivo`.
6. Filtrar por el código DANE resuelto.
7. Obtener los cultivos únicos presentes y ordenar según `CULTIVOS_MVP`.
8. Si no queda ningún cultivo, devolver 404 en lugar de inventar cobertura.
9. Construir la respuesta con el nombre display y el departamento canónicos.

## Interfaz o contrato

### `GET /cultivos/{municipio}`

Respuesta esperada:

```json
{
  "codigo_dane": "73168",
  "municipio": "Chaparral",
  "departamento": "Tolima",
  "cultivos": ["Café", "Cacao", "Maíz"]
}
```

Esquema `CultivosMunicipioResponse`:

| Campo          | Tipo        | Nullable | Regla                          |
| -------------- | ----------- | -------- | ------------------------------ |
| `codigo_dane`  | `str`       | No       | 5 dígitos.                     |
| `municipio`    | `str`       | No       | Nombre display normalizado.    |
| `departamento` | `str`       | No       | Departamento normalizado.      |
| `cultivos`     | `list[str]` | No       | Subconjunto de `CULTIVOS_MVP`. |

## Manejo de errores

| Situación                                  | HTTP | Mensaje sugerido                                           |
| ------------------------------------------ | ---- | ---------------------------------------------------------- |
| Municipio no reconocible                   | 404  | `Municipio no encontrado en el MVP`                        |
| Municipio válido sin cobertura en la tabla | 404  | `No hay cultivos disponibles para el municipio solicitado` |
| Faltan columnas mínimas en la tabla        | 503  | `feature_matrix.parquet no disponible`                     |
| Error interno al leer la tabla             | 500  | `Error interno al resolver cultivos por municipio`         |

## Salida esperada

- Respuesta consistente para el mismo municipio.
- Lista sin duplicados.
- Solo cultivos del MVP.

## Caso feliz

- Dado `municipio = "Chaparral"`.
- Cuando el frontend llama `GET /cultivos/Chaparral`.
- Entonces devuelve los cultivos reales disponibles para Chaparral en el orden del MVP.

## Validación

- Dado un municipio existente del MVP.
- Cuando se consulta el endpoint.
- Entonces la respuesta contiene el código DANE, el nombre display y el departamento correctos, y no incluye cultivos fuera del MVP.

## Criterios de aceptación

- El municipio se resuelve de forma robusta desde nombre o código.
- La cobertura se deriva de la base histórica, no de una lista hardcodeada.
- El resultado no contiene duplicados ni cultivos inválidos.

## Restricciones técnicas

- No usar búsqueda fuzzy.
- No usar base de datos.
- No inventar cultivos para completar listas vacías.
