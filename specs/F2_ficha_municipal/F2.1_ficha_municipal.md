## Agente

- role: Ingeniero frontend Next.js.
- goal: Entregar la ficha municipal con rendimiento historico, riesgo, SHAP y recomendacion.
- backstory: Pantalla clave para que UMATAs entiendan el riesgo y tomen decisiones por cultivo.

## Objetivo

- Mostrar el detalle municipal por cultivo con prediccion de riesgo, grafica historica y explicacion clara.

## Alcance

### Incluye

- Header con municipio, departamento, cultivo y anio objetivo.
- Tarjeta de prediccion con `rendimiento_esperado`, `prob_riesgo_alto` y semaforo.
- Grafica historica de rendimiento (serie anual) con linea de promedio 3 anios.
- Bloque de factores determinantes (SHAP) en formato de barras o ranking.
- Bloque de recomendacion textual segun el riesgo.
- Estados de carga, error parcial y vacio.
- Diseno responsive para desktop y mobile.

### No incluye

- Mapa de Colombia (F1).
- Comparador de cultivos (F3).
- Chat o reportes interactivos (A3.2, A3.3).
- Nuevos endpoints backend.

## Entradas

- Params de ruta: `codigo_dane`, `cultivo`.
- Query param opcional: `anio`.
- `GET /rendimiento/{municipio}/{cultivo}` (A2.1).
- `POST /predecir` (A1.3).
- `GET /reporte/{municipio}/{cultivo}?formato=texto` (A3.3).

## Inputs esperados

### `GET /rendimiento/{municipio}/{cultivo}`

- Response:
  - `codigo_dane`, `municipio`, `departamento`, `cultivo`
  - `n_anios`, `año_min`, `año_max`
  - `serie`: lista de `{ año, rendimiento, area_sembrada, rendimiento_prom3a, tendencia_3a }`

### `POST /predecir`

- Request: `{ municipio, cultivo, año }` (municipio acepta codigo DANE).
- Response: `{ codigo_dane, municipio, departamento, cultivo, año, rendimiento_esperado, prob_riesgo_alto, etiqueta_riesgo }`

### `GET /reporte/{municipio}/{cultivo}?formato=texto`

- Response:
  - `codigo_dane`, `municipio`, `departamento`, `cultivo`
  - `año_referencia`
  - `titulo`
  - `contenido_texto` (string con secciones `=== Titulo ===`)
  - `secciones` (lista de titulos)
  - `fuentes`

## Logica / Proceso sugerido

1. Leer params de ruta y validar `codigo_dane` (5 digitos). Si no es valido, mostrar error y link de regreso.
2. Llamar `GET /rendimiento/{municipio}/{cultivo}` para obtener la serie historica.
3. Resolver `anio_objetivo`:
   - Si viene `anio` en query y es > `año_max`, usarlo.
   - Si no, usar `año_max + 1`.
4. Llamar `POST /predecir` con `{ municipio: codigo_dane, cultivo, año: anio_objetivo }`.
5. Llamar `GET /reporte/{municipio}/{cultivo}?formato=texto` para narrativa y SHAP.
6. Parsear `contenido_texto` en secciones usando el patron `=== {titulo} ===`.
7. Extraer del bloque `Riesgo`:
   - Texto de analisis (lineas posteriores a `Analisis:` si existen).
   - Lineas bajo `Factores determinantes (SHAP):` para construir items.
8. Construir items SHAP con el formato:
   - `label`: texto antes de `:`.
   - `value`: numero si existe (si no, `null`).
   - `direction`: texto dentro de parentesis si existe.
9. Renderizar barras:
   - Si hay `value` numerico, escalar por max abs y renderizar barra proporcional.
   - Si no hay valores, usar barras por ranking (100%, 80%, 60%, 40%, 20%).
10. Extraer del bloque `Recomendacion` el texto para la tarjeta de accion.

## Manejo de errores

| Situacion                             | Comportamiento UI                                        |
| ------------------------------------- | -------------------------------------------------------- |
| `GET /rendimiento` 404                | Mostrar grafica vacia y mensaje "Sin historico".         |
| `POST /predecir` 422 (anio no futuro) | Mostrar error y ofrecer ajustar `anio_objetivo`.         |
| `POST /predecir` 404 (sin datos)      | Mostrar tarjeta de riesgo en estado "Sin datos".         |
| `GET /reporte` 404                    | Ocultar SHAP y recomendacion; mostrar aviso informativo. |
| Timeout o 5xx en alguna llamada       | Banner de error parcial y boton de reintento.            |

## Salida esperada

- Ficha municipal con tarjeta de riesgo, grafica historica y bloque SHAP.
- Recomendacion visible con texto coherente al nivel de riesgo.
- Navegacion clara para volver al mapa.

## Caso feliz

- Dado un `codigo_dane` y `cultivo` validos.
- Cuando la pagina carga.
- Entonces se muestran prediccion, grafica historica y factores SHAP.

## Validacion

- Dado un `anio` invalido o <= `año_max`.
- Cuando se consulta `POST /predecir`.
- Entonces la UI muestra error y ajusta el anio al siguiente disponible.

## Criterios de aceptacion

- La ficha usa endpoints existentes sin hardcodear datos.
- El anio objetivo siempre es futuro respecto al historico.
- Los 5 factores SHAP (si existen) aparecen ordenados.
- La grafica muestra rendimiento historico y promedio 3 anios.
- La UI se ve bien en mobile y desktop.

## Restricciones tecnicas

- Next.js 14 App Router.
- `react-plotly.js` debe cargarse solo en cliente (SSR deshabilitado).
- Usar `NEXT_PUBLIC_API_BASE_URL` para configurar el backend.
- Tipografia no puede ser Inter, Roboto, Arial ni system.
- Definir variables CSS de color y fondo con gradiente sutil.
- Animaciones: 1 carga inicial y 1 transicion suave al cambiar de cultivo.
