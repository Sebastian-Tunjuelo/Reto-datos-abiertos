# C2.4 — Validación de las plantillas de prompt

## Metadatos

| Campo         | Valor                                                                                                                                  |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| ID            | C2.4                                                                                                                                   |
| Módulo o área | `specs/C2_prompts/validate_c2.py`                                                                                                      |
| Depende de    | C2.1, C2.2, C2.3, `modules/conversational/prompts.py`, `modules/conversational/rag.py`                                                |
| Produce para  | — (validación final del bloque C2)                                                                                                     |
| Estado        | ✅ Completo                                                                                                                            |

## Agente

- **role**: Ingeniero de QA / validación de módulo conversacional.
- **goal**: Verificar que las tres plantillas de prompt renderizan correctamente, cubren todos
  los campos del contrato `ContextoRecuperado`, y se comportan de forma predecible ante entradas
  incompletas o inválidas.
- **backstory**: Las plantillas son el contrato entre el módulo de recuperación (C1) y el LLM.
  Si una plantilla falla silenciosamente ante un campo `None`, el LLM recibe un prompt incompleto
  y genera respuestas incorrectas. La validación debe detectar estos casos antes de integrar con F4.

## Objetivo

Implementar el script `specs/C2_prompts/validate_c2.py` que valide las tres plantillas de prompt
end-to-end: renderizado con datos completos, renderizado con datos parciales, comportamiento ante
entradas inválidas y verificación de contratos de retorno.

## Alcance

### Incluye

- Script ejecutable con `python specs/C2_prompts/validate_c2.py` sin levantar la API.
- Verificaciones para C2.1: `build_prompt_conversacional()`, `build_contexto_recuperado()`,
  `format_feature_for_prompt()`, `build_system_prompt()`.
- Verificaciones para C2.2: `build_prompt_reporte_umata()`, `_build_encabezado_umata()`.
- Verificaciones para C2.3: `build_prompt_comparacion_cultivos()`, `_build_tabla_comparativa()`.
- Reporte de cada verificación con prefijo `[PASS]` o `[FAIL]` y descripción accionable.
- Código de salida `0` si todas las verificaciones pasan, `1` si alguna falla.
- Continuación de la ejecución aunque una verificación falle (no abortar en el primer error).

### No incluye

- Llamar al LLM ni a la API de Gemini.
- Leer archivos Parquet del proyecto.
- Verificar la calidad semántica de las respuestas del LLM.
- Verificar el endpoint `/chat` ni ningún endpoint de la API.

## Entradas

- `modules/conversational/prompts.py` (importado directamente).
- Objetos `ContextoRecuperado` construidos en memoria dentro del script (sin leer Parquet).

## Inputs esperados

El script construye sus propios fixtures en memoria:

```python

ctx_completo = ContextoRecuperado()
```

## Lógica / Proceso sugerido

### Bloque 1 — Verificaciones de C2.1 (prompt conversacional)

1. Verificar que `build_system_prompt("campesino")` retorna string con longitud ≥ 100.
2. Verificar que `build_system_prompt("institucional")` retorna string con longitud ≥ 100.
3. Verificar que `build_system_prompt("desconocido")` retorna el system prompt campesino (fallback).
4. Verificar que `build_contexto_recuperado(None, None, None, None)` retorna string no vacío
   con el placeholder de contexto no disponible.
5. Verificar que `build_contexto_recuperado(prediccion=ctx_completo.prediccion, tono="campesino")`
   retorna string que contiene `"1.45"` o `"1,45"` (el rendimiento esperado).
6. Verificar que `format_feature_for_prompt({}, "campesino")` retorna string no vacío sin lanzar excepción.
7. Verificar que `format_feature_for_prompt({"feature": "prec_acum_mm", "valor": 320.5}, "campesino")`
   retorna string que contiene `"320"`.
8. Verificar que `build_prompt_conversacional(ctx_completo, "¿Cuál es el riesgo?", "campesino")`
   retorna `dict` con claves `"system"` y `"user"`, ambas strings no vacíos.
9. Verificar que `build_prompt_conversacional(ctx_vacio, "", "campesino")` retorna `dict` válido
   sin lanzar excepción.
10. Verificar que el `"user"` de `build_prompt_conversacional(ctx_completo, ...)` contiene
    la narrativa del fixture.

### Bloque 2 — Verificaciones de C2.2 (reporte UMATA)

1. Verificar que `build_prompt_reporte_umata(ctx_completo, "Chaparral", "Café", 2024)` retorna
   `dict` con claves `"system"` y `"user"`.
2. Verificar que el `"user"` contiene `"Chaparral"` y `"Tolima"` (departamento obtenido de `dane_codes`).
3. Verificar que el `"user"` contiene `"Café"` y `"2024"`.
4. Verificar que el `"system"` contiene las 8 secciones obligatorias del reporte UMATA
   (buscar al menos 5 de los 8 encabezados de sección en el texto).
5. Verificar que `build_prompt_reporte_umata(ctx_vacio, None, None, None)` retorna `dict` válido
   sin lanzar excepción.
6. Verificar que el `"user"` de `build_prompt_reporte_umata(ctx_vacio, ...)` contiene
   `"Predicción no disponible"` o `"no disponible"` (case-insensitive).
7. Verificar que `build_prompt_reporte_umata(ctx_completo, "MunicipioInexistente", "Café", 2024)`
   retorna `dict` válido sin lanzar excepción (municipio no en `dane_codes`).

### Bloque 3 — Verificaciones de C2.3 (comparación de cultivos)

1. Verificar que `build_prompt_comparacion_cultivos({"Café": ctx_completo}, "Chaparral", 2024)`
   retorna `dict` con claves `"system"` y `"user"`.
2. Verificar que el `"user"` contiene `"Café"`.
3. Verificar que el `"user"` contiene una nota indicando que Cacao y Maíz no tienen datos.
4. Verificar que `build_prompt_comparacion_cultivos({"Café": ctx_completo, "Cacao": ctx_completo, "Maíz": ctx_completo}, "Chaparral", 2024)`
   retorna `dict` con `"user"` que contiene los tres cultivos.
5. Verificar que `build_prompt_comparacion_cultivos({}, "Chaparral", 2024)` lanza `ValueError`
   con mensaje que contiene `"[C2.3]"`.
6. Verificar que `build_prompt_comparacion_cultivos({"Café": ctx_vacio}, "Chaparral", 2024)`
   retorna `dict` válido sin lanzar excepción (ctx con prediccion=None).
7. Verificar que el `"system"` de comparación contiene las 4 secciones obligatorias del ranking.

### Bloque 4 — Verificaciones de compatibilidad con `chat_engine.py`

1. Verificar que `build_system_prompt` acepta exactamente los parámetros `(tono: str = "campesino")`.
2. Verificar que `build_user_prompt` acepta exactamente los parámetros
   `(pregunta, contexto, contexto_recuperado, glosario_relevante="")`.
3. Verificar que `build_contexto_recuperado` acepta exactamente los parámetros
   `(prediccion=None, narrativa=None, top_features=None, serie_historica=None, tono="campesino")`.
4. Verificar que `format_feature_for_prompt` acepta exactamente los parámetros
   `(feature: dict, tono: str = "campesino")`.

## Manejo de errores

| Situación                                              | Comportamiento                                                                                                    |
| ------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------- |
| Una verificación lanza excepción inesperada            | Capturar con `try/except`, reportar `[FAIL] {nombre}: excepción inesperada: {e}` y continuar                     |
| `prompts.py` no importa correctamente                  | El script falla con `ImportError` y mensaje claro antes de ejecutar verificaciones                                |
| `ContextoRecuperado` no importa correctamente          | El script falla con `ImportError` y mensaje claro antes de ejecutar verificaciones                                |
| Una función nueva no existe en `prompts.py`            | Reportar `[FAIL] {nombre}: función no encontrada en prompts.py` y continuar con las demás verificaciones         |

## Salida esperada

- Cada verificación imprime `[PASS] {ID} {descripción}` o `[FAIL] {ID} {descripción}: {detalle}`.
- Al final: `Resultado: {n_pass}/{n_total} verificaciones pasaron`.
- Código de salida `0` si todas pasan, `1` si alguna falla.
- El script no requiere datos externos ni conexión a red.

## Caso feliz

- Dado que C2.1, C2.2 y C2.3 están implementados correctamente.
- Cuando se ejecuta `python specs/C2_prompts/validate_c2.py`.
- Entonces todas las verificaciones reportan `[PASS]` y el código de salida es `0`.

## Validación

- Dado que `build_prompt_reporte_umata()` no está implementada en `prompts.py`.
- Cuando se ejecuta `python specs/C2_prompts/validate_c2.py`.
- Entonces las verificaciones del Bloque 2 reportan `[FAIL]`, las demás continúan ejecutándose,
  y el código de salida es `1`.

## Criterios de aceptación

1. El script es ejecutable con `python specs/C2_prompts/validate_c2.py` sin levantar la API.
2. Cada verificación reporta `[PASS]` o `[FAIL]` con descripción accionable.
3. El script continúa ejecutando verificaciones aunque una falle.
4. Código de salida `0` si todas pasan, `1` si alguna falla.
5. El script cubre los 4 bloques: C2.1 (conversacional), C2.2 (UMATA), C2.3 (comparación) y compatibilidad.
6. El script no llama al LLM ni lee archivos Parquet.
7. El script no requiere conexión a red ni variables de entorno.
8. Las verificaciones de compatibilidad confirman que las firmas de las funciones existentes
   no fueron modificadas respecto al contrato de A3.2.

```python
validate_c2.py — Validación de las plantillas de prompt C2.

Ejecutar desde la raíz del proyecto:
    python specs/C2_prompts/validate_c2.py
"""

import logging
from typing import Optional

try:
    from modules.conversational.prompts import (
        build_system_prompt,
        build_user_prompt,
        build_contexto_recuperado,
        format_feature_for_prompt,

    )

except ImportError as e:
    print(f"[ERROR] No se pudo importar el módulo: {e}")
    sys.exit(1)

logging.disable(logging.NOTSET)


def _check(nombre: str, condicion: bool, detalle: str = "") -> bool:
    estado = "[PASS]" if condicion else "[FAIL]"
    msg = f"{estado} {nombre}"
    if not condicion and detalle:
        msg += f": {detalle}"
    print(msg)
    return condicion


def _build_ctx_completo() -> ContextoRecuperado:
    """Construye el fixture completo para las verificaciones."""
    ctx = ContextoRecuperado()
    ctx.prediccion = {
        "rendimiento_esperado": 1.45,
        "etiqueta_riesgo": "Alto",
        "prob_riesgo_alto": 0.71
    }
    ctx.narrativa = "El déficit de lluvia acumulada y el aumento del índice de fertilizantes elevan el riesgo."
    ctx.top_features = [
        {"feature": "prec_acum_mm", "nombre_amigable": "Precipitación acumulada",
         "valor": 320.5, "importancia": 0.42, "direccion": "aumenta riesgo"},
        {"feature": "indice_agroinsumos", "nombre_amigable": "Índice de agroinsumos",

        {"año": 2022, "rendimiento": 1.30},
        {"año": 2023, "rendimiento": 1.45},

    ]
    ctx.fuentes = ["predicciones_con_explicacion.parquet", "narrativa_riesgo"]
    ctx.contexto_efectivo = {"municipio": "Chaparral", "cultivo": "Café", "año": 2024, "escenario": "base"}

    """Construye el fixture vacío para las verificaciones."""
    ctx = ContextoRecuperado()
    ctx.contexto_efectivo = {}
    return ctx
```
