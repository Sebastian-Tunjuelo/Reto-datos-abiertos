# C2.2 — Plantilla de prompt para reporte institucional UMATA

## Metadatos

| Campo         | Valor                                                                                                                                  |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| ID            | C2.2                                                                                                                                   |
| Módulo o área | `modules/conversational/prompts.py`                                                                                                    |
| Depende de    | C2.1, `modules/conversational/rag.py` (contrato `ContextoRecuperado`), `shared/dane_codes.py`                                         |
| Produce para  | `modules/conversational/reports.py`, C3, C2.4                                                                                         |
| Estado        | ✅ Completo                                                                                                                            |

## Agente

- **role**: Ingeniero de prompts / módulo conversacional.
- **goal**: Crear una plantilla de prompt que instruya al LLM para generar un reporte institucional
  estructurado, listo para ser presentado por una UMATA o secretaría de agricultura a tomadores
  de decisión.
- **backstory**: Los técnicos de UMATA necesitan un documento con secciones fijas, lenguaje formal,
  datos verificables y recomendaciones accionables. El reporte debe poder generarse a partir del
  mismo `ContextoRecuperado` que usa el chat, sin requerir datos adicionales.

## Objetivo

Implementar la función `build_prompt_reporte_umata(contexto_recuperado, municipio, cultivo, año)`
en `modules/conversational/prompts.py`, que construya el prompt completo (system + user) para
que el LLM genere un reporte institucional estructurado con secciones fijas.

## Alcance

### Incluye

- Función `build_prompt_reporte_umata(contexto_recuperado, municipio, cultivo, año)` que retorna
  `dict` con claves `"system"` y `"user"`.
- System prompt institucional específico para reportes UMATA (distinto al de chat institucional):
  instruye al LLM a generar un documento con secciones fijas, no una respuesta conversacional.
- User prompt que inyecta todos los datos disponibles del `ContextoRecuperado` en formato
  estructurado para que el LLM los use como base del reporte.
- Secciones obligatorias del reporte que el LLM debe generar:
  1. Encabezado (municipio, departamento, cultivo, año, fecha de generación).
  2. Resumen ejecutivo (máximo 3 oraciones).
  3. Predicción de rendimiento y nivel de riesgo.
  4. Factores determinantes (top features SHAP).
  5. Análisis de riesgo (narrativa M4).
  6. Serie histórica de rendimiento (últimos 3 años si disponible).
  7. Recomendaciones accionables (mínimo 2, máximo 4).
  8. Fuentes de datos utilizadas.
- Función auxiliar `_build_encabezado_umata(municipio, cultivo, año)` que construye el encabezado
  usando `shared/dane_codes.py` para obtener el departamento.

### No incluye

- Generación del PDF ni del archivo de reporte — eso es responsabilidad de `reports.py` (C3).
- Llamar al LLM desde `prompts.py`.
- Leer archivos Parquet desde `prompts.py`.
- Modificar las funciones existentes de C2.1.

## Entradas

- `contexto_recuperado`: instancia de `ContextoRecuperado` con los campos del contrato global.
- `municipio`: string con el nombre display del municipio (Title Case). Puede ser `None`.
- `cultivo`: string con el nombre del cultivo (`Café`, `Cacao` o `Maíz`). Puede ser `None`.
- `año`: entero con el año de referencia. Puede ser `None`.

## Inputs esperados

| Parámetro             | Tipo                 | Nullable | Regla                                                                                                                    |
| --------------------- | -------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------ |
| `contexto_recuperado` | `ContextoRecuperado` | No       | Objeto con campos opcionales internos. Nunca `None` el objeto en sí.                                                     |
| `municipio`           | `str`                | Sí       | Si es `None`, usar `"Municipio no especificado"` en el encabezado.                                                       |
| `cultivo`             | `str`                | Sí       | Si es `None`, usar `"Cultivo no especificado"` en el encabezado.                                                         |
| `año`                 | `int`                | Sí       | Si es `None`, usar `"Año no especificado"` en el encabezado.                                                             |
| `prediccion`          | `dict` (interno)     | Sí       | Si es `None`, la sección de predicción debe indicar `"Predicción no disponible"`.                                        |
| `narrativa`           | `str` (interno)      | Sí       | Si es `None`, la sección de análisis debe indicar `"Análisis narrativo no disponible"`.                                  |
| `top_features`        | `list` (interno)     | Sí       | Si es `None` o `[]`, la sección de factores debe indicar `"Factores SHAP no disponibles"`.                               |
| `serie_historica`     | `list` (interno)     | Sí       | Si es `None` o `[]`, la sección histórica debe indicar `"Serie histórica no disponible"`.                                |

## Lógica / Proceso sugerido

### `_build_encabezado_umata(municipio, cultivo, año)` ← función auxiliar interna

1. Importar `get_codigo`, `DANE_TO_DEPT` de `shared/dane_codes.py`.
2. Si `municipio` no es `None`, obtener el código DANE con `get_codigo(municipio)`.
3. Si el código existe, obtener el departamento con `DANE_TO_DEPT.get(codigo, "Departamento no disponible")`.
4. Si `municipio` es `None`, usar `"Municipio no especificado"` y `"Departamento no disponible"`.
5. Retornar un string con el encabezado formateado:
   ```
   MUNICIPIO: {municipio}
   DEPARTAMENTO: {departamento}
   CULTIVO: {cultivo}
   AÑO DE REFERENCIA: {año}
   GENERADO POR: SiembraSegura IA
   ```

### `build_prompt_reporte_umata(contexto_recuperado, municipio, cultivo, año)` ← función nueva

1. Construir el system prompt institucional para reportes (ver sección de interfaz).
2. Llamar a `_build_encabezado_umata(municipio, cultivo, año)` para el encabezado.
3. Construir la sección de predicción:
   - Si `prediccion` no es `None`: incluir `rendimiento_esperado` y `etiqueta_riesgo`.
   - Si `prediccion` es `None`: incluir `"Predicción no disponible"`.
4. Construir la sección de factores SHAP:
   - Si `top_features` no es `None` y tiene elementos: listar hasta 5 features con nombre y valor.
   - Si no: incluir `"Factores SHAP no disponibles"`.
5. Construir la sección de narrativa:
   - Si `narrativa` no es `None`: incluir el texto completo.
   - Si no: incluir `"Análisis narrativo no disponible"`.
6. Construir la sección histórica:
   - Si `serie_historica` no es `None` y tiene elementos: listar los últimos 3 años con rendimiento.
   - Si no: incluir `"Serie histórica no disponible"`.
7. Construir la sección de fuentes a partir de `contexto_recuperado.fuentes`.
8. Ensamblar el user prompt con todas las secciones y las instrucciones al LLM para generar
   el reporte con las 8 secciones obligatorias.
9. Retornar `{"system": system_prompt, "user": user_prompt}`.

## Interfaz o contrato

```python
# Función auxiliar interna (prefijo _)
def _build_encabezado_umata(
    municipio: Optional[str],
    cultivo: Optional[str],
    año: Optional[int]
) -> str:
    """
    Construye el encabezado institucional del reporte UMATA.
    Usa shared/dane_codes.py para obtener el departamento.
    No lanza excepción si municipio es None.
    """

# Función pública nueva
def build_prompt_reporte_umata(
    contexto_recuperado: "ContextoRecuperado",
    municipio: Optional[str] = None,
    cultivo: Optional[str] = None,
    año: Optional[int] = None
) -> dict:
    """
    Construye el prompt completo para generar un reporte institucional UMATA.

    Returns:
        dict con claves "system" y "user", ambas strings no vacíos.
        El system prompt instruye al LLM a generar un documento estructurado.
        El user prompt inyecta todos los datos disponibles del contexto.
    """
```

### System prompt para reporte UMATA

El system prompt debe instruir al LLM con las siguientes reglas:

```
Eres un asistente técnico de SiembraSegura IA especializado en generar reportes institucionales
para UMATAs y secretarías de agricultura de Colombia.

Tu tarea es generar un reporte técnico estructurado con las siguientes secciones obligatorias:
1. ENCABEZADO (municipio, departamento, cultivo, año, generado por)
2. RESUMEN EJECUTIVO (máximo 3 oraciones, lenguaje formal)
3. PREDICCIÓN DE RENDIMIENTO Y RIESGO (datos numéricos y clasificación)
4. FACTORES DETERMINANTES (factores SHAP con nombre técnico y valor)
5. ANÁLISIS DE RIESGO (narrativa técnica)
6. SERIE HISTÓRICA (tabla de rendimiento por año)
7. RECOMENDACIONES (mínimo 2, máximo 4, accionables y específicas)
8. FUENTES DE DATOS (artefactos utilizados)

REGLAS ESTRICTAS:
- Solo usa los datos proporcionados en el CONTEXTO. No inventes cifras.
- Si una sección no tiene datos, escribe "Información no disponible" en esa sección.
- Usa lenguaje técnico formal apropiado para un documento oficial.
- Las recomendaciones deben ser específicas para el municipio y cultivo indicados.
- No incluyas secciones adicionales fuera de las 8 obligatorias.
- El reporte debe poder leerse como un documento oficial independiente.
```

### User prompt — estructura de datos inyectados

```
DATOS PARA EL REPORTE:

{encabezado}

PREDICCIÓN:
{datos_prediccion}

FACTORES DETERMINANTES (SHAP):
{datos_features}

ANÁLISIS DE RIESGO:
{narrativa}

SERIE HISTÓRICA:
{serie_historica}

FUENTES UTILIZADAS:
{fuentes}

---
Genera el reporte institucional completo con las 8 secciones obligatorias.
```

## Manejo de errores

| Situación                                    | Comportamiento                                                                                                  |
| -------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| `municipio` es `None`                        | Usar `"Municipio no especificado"` en encabezado, no lanzar excepción                                           |
| `cultivo` es `None`                          | Usar `"Cultivo no especificado"` en encabezado, no lanzar excepción                                             |
| `año` es `None`                              | Usar `"Año no especificado"` en encabezado, no lanzar excepción                                                 |
| `prediccion` es `None`                       | Incluir `"Predicción no disponible"` en la sección correspondiente                                              |
| `top_features` es `None` o `[]`              | Incluir `"Factores SHAP no disponibles"` en la sección correspondiente                                          |
| `narrativa` es `None`                        | Incluir `"Análisis narrativo no disponible"` en la sección correspondiente                                      |
| `serie_historica` es `None` o `[]`           | Incluir `"Serie histórica no disponible"` en la sección correspondiente                                         |
| `get_codigo(municipio)` retorna `None`       | `logger.warning("[C2.2] Municipio '{municipio}' no encontrado en dane_codes")` → usar `"Departamento no disponible"` |

## Salida esperada

- `build_prompt_reporte_umata(ctx, "Chaparral", "Café", 2024)` → `dict` con claves `"system"` y `"user"`.
- `"system"` contiene las instrucciones para generar el reporte con las 8 secciones.
- `"user"` contiene el encabezado con `MUNICIPIO: Chaparral`, `DEPARTAMENTO: Tolima`, `CULTIVO: Café`.
- `"user"` contiene los datos de predicción, features, narrativa y serie histórica si están disponibles.
- `build_prompt_reporte_umata(ctx_vacio, None, None, None)` → `dict` válido sin lanzar excepción.

## Caso feliz

- Dado un `ContextoRecuperado` con `prediccion`, `narrativa`, `top_features` y `serie_historica` no nulos.
- Cuando se llama a `build_prompt_reporte_umata(ctx, "Chaparral", "Café", 2024)`.
- Entonces retorna un `dict` con `"system"` y `"user"` no vacíos, donde:
  - `"user"` contiene `MUNICIPIO: Chaparral` y `DEPARTAMENTO: Tolima`.
  - `"user"` contiene el rendimiento esperado y la etiqueta de riesgo.
  - `"user"` contiene al menos un factor SHAP.
  - `"user"` contiene la narrativa de riesgo.

## Validación

- Dado un `ContextoRecuperado` con todos los campos internos `None`.
- Cuando se llama a `build_prompt_reporte_umata(ctx_vacio, None, None, None)`.
- Entonces retorna un `dict` válido sin lanzar excepción, con `"user"` que contiene
  `"Predicción no disponible"`, `"Factores SHAP no disponibles"` y `"Análisis narrativo no disponible"`.

## Criterios de aceptación

1. `build_prompt_reporte_umata()` retorna `dict` con claves `"system"` y `"user"`.
2. El system prompt contiene las instrucciones para las 8 secciones obligatorias.
3. El user prompt incluye el encabezado con municipio, departamento (obtenido de `dane_codes.py`), cultivo y año.
4. Ninguna función lanza excepción cuando todos los campos opcionales son `None`.
5. Cuando `municipio` es un municipio MVP válido, el departamento se obtiene correctamente de `shared/dane_codes.py`.
6. Cuando `municipio` no está en `dane_codes`, se registra un warning y se usa `"Departamento no disponible"`.
7. La función no llama al LLM ni lee archivos Parquet.
8. Las firmas de las funciones existentes de C2.1 no se modifican.
