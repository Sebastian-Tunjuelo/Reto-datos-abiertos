# C2.3 — Plantilla de prompt para comparación de cultivos

## Metadatos

| Campo         | Valor                                                                                                                                                                    |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| ID            | C2.3                                                                                                                                                                     |
| Módulo o área | `modules/conversational/prompts.py`                                                                                                                                      |
| Depende de    | C2.1, `modules/conversational/rag.py` (contrato `ContextoRecuperado`), `shared/config.py` (`CULTIVOS_MVP`), `shared/dane_codes.py`                                       |
| Produce para  | `modules/conversational/chat_engine.py`, C2.4                                                                                                                            |
| Estado        | ✅ Completo                                                                                                                                                              |

## Agente

- **role**: Ingeniero de prompts / módulo conversacional.
- **goal**: Crear una plantilla de prompt que instruya al LLM para generar un ranking comparativo
  de los tres cultivos MVP (Café, Cacao, Maíz) para un municipio dado, usando los datos de
  predicción, riesgo y aptitud disponibles en el contexto recuperado.
- **backstory**: Los extensionistas necesitan una respuesta que compare los tres cultivos en un
  mismo municipio y recomiende cuál sembrar según rendimiento esperado, nivel de riesgo y aptitud
  UPRA. El contexto puede tener datos para uno, dos o los tres cultivos.

## Objetivo

Implementar la función `build_prompt_comparacion_cultivos(contextos_por_cultivo, municipio, año)`
en `modules/conversational/prompts.py`, que construya el prompt completo (system + user) para
que el LLM genere un ranking comparativo de los cultivos MVP con justificación técnica.

## Alcance

### Incluye

- Función `build_prompt_comparacion_cultivos(contextos_por_cultivo, municipio, año)` que recibe
  un diccionario `{cultivo: ContextoRecuperado}` y retorna `dict` con claves `"system"` y `"user"`.
- System prompt específico para comparación de cultivos: instruye al LLM a generar un ranking
  ordenado con justificación, no una respuesta conversacional libre.
- User prompt que inyecta los datos de cada cultivo disponible en formato tabular comparativo.
- Función auxiliar `_build_tabla_comparativa(contextos_por_cultivo)` que construye la tabla
  de datos para los cultivos disponibles.
- Manejo del caso en que solo hay datos para uno o dos cultivos: el LLM debe indicar cuáles
  cultivos no tienen datos disponibles y basar el ranking solo en los que sí tienen.
- Uso de `CULTIVOS_MVP` de `shared/config.py` para determinar qué cultivos deberían estar
  presentes y cuáles faltan.

### No incluye

- Llamar al LLM desde `prompts.py`.
- Leer archivos Parquet desde `prompts.py`.
- Calcular el ranking directamente en Python — eso lo hace el LLM con los datos inyectados.
- Modificar las funciones existentes de C2.1 ni C2.2.
- Implementar el endpoint de comparación — eso es responsabilidad del orquestador.

## Entradas

- `contextos_por_cultivo`: `dict[str, ContextoRecuperado]` donde la clave es el nombre del cultivo
  (`"Café"`, `"Cacao"`, `"Maíz"`). Puede tener 1, 2 o 3 entradas. No puede ser `None` ni vacío.
- `municipio`: string con el nombre display del municipio. Puede ser `None`.
- `año`: entero con el año de referencia. Puede ser `None`.

## Inputs esperados

| Parámetro               | Tipo                              | Nullable | Regla                                                                                                                                    |
| ----------------------- | --------------------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `contextos_por_cultivo` | `dict[str, ContextoRecuperado]`   | No       | Al menos una entrada. Si está vacío, lanzar `ValueError("[C2.3] contextos_por_cultivo no puede estar vacío")`.                           |
| `municipio`             | `str`                             | Sí       | Si es `None`, usar `"Municipio no especificado"` en el prompt.                                                                           |
| `año`                   | `int`                             | Sí       | Si es `None`, usar `"Año no especificado"` en el prompt.                                                                                 |
| `ContextoRecuperado`    | objeto (por cultivo)              | No       | Cada valor del dict es un `ContextoRecuperado`. Sus campos internos pueden ser `None`.                                                   |
| `prediccion` (interno)  | `dict`                            | Sí       | Si es `None` para un cultivo, ese cultivo aparece en la tabla con `"Sin datos"`.                                                         |
| `top_features` (interno)| `list`                            | Sí       | Si es `None` o `[]`, omitir la columna de factores para ese cultivo.                                                                     |

## Lógica / Proceso sugerido

### `_build_tabla_comparativa(contextos_por_cultivo)` ← función auxiliar interna

1. Importar `CULTIVOS_MVP` de `shared/config.py`.
2. Inicializar lista de filas para la tabla.
3. Para cada cultivo en `CULTIVOS_MVP`:
   - Si el cultivo está en `contextos_por_cultivo`:
     - Extraer `rendimiento_esperado` de `prediccion` (o `"N/D"` si `None`).
     - Extraer `etiqueta_riesgo` de `prediccion` (o `"N/D"` si `None`).
     - Extraer el primer factor SHAP de `top_features` (o `"N/D"` si `None` o vacío).
     - Agregar fila: `{cultivo} | {rendimiento} t/ha | Riesgo: {etiqueta} | Factor principal: {factor}`.
   - Si el cultivo NO está en `contextos_por_cultivo`:
     - Agregar fila: `{cultivo} | Sin datos disponibles`.
4. Retornar el string con la tabla formateada.

### `build_prompt_comparacion_cultivos(contextos_por_cultivo, municipio, año)` ← función nueva

1. Validar que `contextos_por_cultivo` no sea `None` ni vacío. Si lo es, lanzar `ValueError`.
2. Construir el system prompt para comparación (ver sección de interfaz).
3. Construir el encabezado del user prompt con municipio y año.
4. Llamar a `_build_tabla_comparativa(contextos_por_cultivo)` para obtener la tabla de datos.
5. Determinar qué cultivos tienen datos y cuáles no, usando `CULTIVOS_MVP`.
6. Si hay cultivos sin datos, incluir una nota en el user prompt indicando cuáles faltan.
7. Incluir las narrativas disponibles por cultivo (máximo 2 oraciones por cultivo).
8. Ensamblar el user prompt con la tabla, las notas y las instrucciones al LLM.
9. Retornar `{"system": system_prompt, "user": user_prompt}`.

## Interfaz o contrato

```python
# Función auxiliar interna (prefijo _)
def _build_tabla_comparativa(

    """

# Función pública nueva
def build_prompt_comparacion_cultivos(
    contextos_por_cultivo: dict,  # {cultivo: ContextoRecuperado}
    municipio: Optional[str] = None,
    año: Optional[int] = None
) -> dict:
    """
    Construye el prompt completo para generar un ranking comparativo de cultivos.

        municipio: Nombre display del municipio. Puede ser None.

        dict con claves "system" y "user", ambas strings no vacíos.

    Raises:
```

### System prompt para comparación de cultivos

```
Eres un asistente técnico de SiembraSegura IA especializado en comparar cultivos agrícolas
para extensionistas y técnicos de UMATA en Colombia.

Tu tarea es analizar los datos proporcionados y generar un ranking de cultivos con la siguiente
estructura obligatoria:

1. RANKING (ordenado de mejor a peor opción para el municipio)
   - Posición, cultivo, rendimiento esperado, nivel de riesgo
2. JUSTIFICACIÓN POR CULTIVO (máximo 2 oraciones por cultivo)
3. RECOMENDACIÓN FINAL (1 cultivo recomendado con razón principal)
4. ADVERTENCIAS (si algún cultivo no tiene datos suficientes)

REGLAS ESTRICTAS:
- Solo usa los datos proporcionados. No inventes rendimientos ni niveles de riesgo.
- Si un cultivo no tiene datos, indícalo en la sección de ADVERTENCIAS y exclúyelo del ranking.
- El ranking debe basarse en: rendimiento esperado (peso alto), nivel de riesgo (peso alto)
  y factor SHAP principal (peso medio).
- Usa lenguaje técnico pero accesible para un extensionista.
- No incluyas secciones adicionales fuera de las 4 obligatorias.
- Si solo hay datos para un cultivo, genera el ranking con ese único cultivo e indica
  que los demás no tienen datos disponibles.
```

### User prompt — estructura de datos inyectados

```
COMPARACIÓN DE CULTIVOS PARA {municipio} — AÑO {año}

DATOS DISPONIBLES:
{tabla_comparativa}

{nota_cultivos_faltantes}

NARRATIVAS DE RIESGO:
{narrativas_por_cultivo}

---
Genera el ranking comparativo con las 4 secciones obligatorias.
```

## Manejo de errores

| Situación                                         | Comportamiento                                                                                                                    |
| ------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `contextos_por_cultivo` es `None` o `{}`          | Lanzar `ValueError("[C2.3] contextos_por_cultivo no puede estar vacío")`                                                          |
| Un cultivo del dict no está en `CULTIVOS_MVP`     | `logger.warning("[C2.3] Cultivo '{cultivo}' no está en CULTIVOS_MVP, se incluye de todas formas")` → incluirlo igual              |
| `prediccion` es `None` para un cultivo            | Mostrar `"Sin datos"` en la tabla para ese cultivo, no lanzar excepción                                                           |
| `municipio` es `None`                             | Usar `"Municipio no especificado"` en el prompt, no lanzar excepción                                                              |
| `año` es `None`                                   | Usar `"Año no especificado"` en el prompt, no lanzar excepción                                                                    |
| Solo 1 cultivo con datos de 3 posibles            | Incluir nota en el user prompt: `"Nota: Solo se dispone de datos para {cultivo}. Café, Cacao y Maíz no tienen datos disponibles."` |

## Salida esperada

- `build_prompt_comparacion_cultivos({"Café": ctx_cafe, "Cacao": ctx_cacao, "Maíz": ctx_maiz}, "Chaparral", 2024)` → `dict` con `"system"` y `"user"`.
- `"user"` contiene la tabla con los tres cultivos, sus rendimientos y niveles de riesgo.
- `"user"` contiene las narrativas disponibles por cultivo.
- `build_prompt_comparacion_cultivos({"Café": ctx_cafe}, "Chaparral", 2024)` → `dict` válido con nota indicando que Cacao y Maíz no tienen datos.
- `build_prompt_comparacion_cultivos({}, "Chaparral", 2024)` → lanza `ValueError`.

## Caso feliz

- Dado un dict con `ContextoRecuperado` para los tres cultivos MVP, todos con `prediccion` no nula.
- Cuando se llama a `build_prompt_comparacion_cultivos(contextos, "Chaparral", 2024)`.
- Entonces retorna un `dict` con `"system"` y `"user"` no vacíos, donde:
  - `"user"` contiene una tabla con Café, Cacao y Maíz.
  - `"user"` contiene el rendimiento esperado y la etiqueta de riesgo de cada cultivo.
  - `"user"` no contiene la nota de cultivos faltantes.

## Validación

- Dado un dict con solo `ContextoRecuperado` para Café.
- Cuando se llama a `build_prompt_comparacion_cultivos({"Café": ctx_cafe}, "Chaparral", 2024)`.
- Entonces retorna un `dict` válido sin lanzar excepción, con `"user"` que contiene
  la nota indicando que Cacao y Maíz no tienen datos disponibles.

- Dado un dict vacío `{}`.
- Cuando se llama a `build_prompt_comparacion_cultivos({}, "Chaparral", 2024)`.
- Entonces lanza `ValueError` con el mensaje `"[C2.3] contextos_por_cultivo no puede estar vacío"`.

## Criterios de aceptación

1. `build_prompt_comparacion_cultivos()` retorna `dict` con claves `"system"` y `"user"`.
2. El system prompt contiene las instrucciones para las 4 secciones obligatorias del ranking.
3. El user prompt incluye la tabla comparativa con los cultivos disponibles.
4. Cuando faltan cultivos, el user prompt incluye una nota explícita indicando cuáles faltan.
5. `_build_tabla_comparativa()` usa `CULTIVOS_MVP` de `shared/config.py` para determinar cultivos faltantes.
6. Un dict vacío lanza `ValueError` con mensaje que incluye `"[C2.3]"`.
7. Un cultivo con `prediccion = None` aparece en la tabla con `"Sin datos"` sin lanzar excepción.
8. La función no llama al LLM ni lee archivos Parquet.
9. Las firmas de las funciones existentes de C2.1 y C2.2 no se modifican.
