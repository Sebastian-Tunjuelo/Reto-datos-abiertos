# C2.1 — Plantilla de prompt conversacional (lenguaje campesino e institucional)

## Agente

- **role**: Ingeniero de prompts / módulo conversacional.
- **goal**: Formalizar las plantillas de prompt conversacional para que el `ChatEngine` pueda
  generar respuestas grounded, adaptadas al tono del usuario y tolerantes a contexto incompleto.
- **backstory**: El `prompts.py` actual tiene una implementación funcional de A3.2 pero sin
  contrato explícito, sin manejo de campos faltantes documentado y sin separación clara entre
  el system prompt y la construcción del contexto. C2.1 formaliza ese contrato.

## Objetivo

Refactorizar y documentar las funciones de plantilla conversacional en `modules/conversational/prompts.py`
para los dos tonos del MVP (campesino e institucional), con contratos explícitos, tolerancia a `None`
en todos los campos opcionales y sin romper las firmas que ya consume `chat_engine.py`.

## Alcance

### Incluye

- Formalizar `build_system_prompt(tono)` con docstring completo y comportamiento ante tono desconocido.
- Formalizar `build_user_prompt(pregunta, contexto, contexto_recuperado, glosario_relevante)` con
  manejo explícito de campos vacíos o `None`.
- Formalizar `build_contexto_recuperado(prediccion, narrativa, top_features, serie_historica, tono)`
  con tolerancia a cualquier campo `None` y sin lanzar excepción.
- Formalizar `format_feature_for_prompt(feature, tono)` con manejo de claves faltantes.
- Agregar la función `build_prompt_conversacional(contexto_recuperado, pregunta, tono)` como
  punto de entrada unificado que orquesta las cuatro funciones anteriores y retorna el prompt
  completo listo para enviar al LLM.
- Documentar en el módulo qué funciones son públicas (consumidas por `chat_engine.py`) y cuáles
  son auxiliares internas.

### No incluye

- Cambiar la firma de `build_system_prompt()`, `build_user_prompt()`, `build_contexto_recuperado()`
  ni `format_feature_for_prompt()` — son contratos públicos de A3.2.
- Llamar a Gemini ni a ningún LLM desde `prompts.py`.
- Leer archivos Parquet ni acceder a `shared/` directamente desde `prompts.py`.
- Implementar la plantilla UMATA (C2.2) ni la de comparación de cultivos (C2.3).

## Entradas

- `contexto_recuperado`: instancia de `ContextoRecuperado` con los campos del contrato global.
- `pregunta`: string con la pregunta del usuario (puede ser vacía — la función debe tolerarlo).
- `tono`: `"campesino"` o `"institucional"`. Si se recibe otro valor, usar `"campesino"` como
  fallback y registrar un warning.

## Inputs esperados

| Parámetro             | Tipo                    | Nullable | Regla                                                                                                  |
| --------------------- | ----------------------- | -------- | ------------------------------------------------------------------------------------------------------ |
| `contexto_recuperado` | `ContextoRecuperado`    | No       | Objeto con campos opcionales internos. Nunca `None` el objeto en sí.                                   |
| `pregunta`            | `str`                   | Sí       | Si es `None` o vacía, usar `"Sin pregunta especificada"` como placeholder.                             |
| `tono`                | `str`                   | Sí       | `"campesino"` o `"institucional"`. Fallback a `"campesino"` si el valor no es reconocido.              |
| `prediccion`          | `dict` (en `build_contexto_recuperado`) | Sí | Puede ser `None`. Si es `None`, omitir la sección de predicción del contexto.    |
| `narrativa`           | `str`                   | Sí       | Puede ser `None`. Si es `None`, omitir la sección de narrativa.                                        |
| `top_features`        | `list`                  | Sí       | Puede ser `None` o lista vacía. Si es `None` o `[]`, omitir la sección de factores.                   |
| `serie_historica`     | `list`                  | Sí       | Puede ser `None` o lista vacía. Si es `None` o `[]`, omitir la sección histórica.                     |

## Lógica / Proceso sugerido

### `build_system_prompt(tono)`

1. Si `tono == "institucional"`, retornar `SYSTEM_PROMPT_INSTITUCIONAL`.
2. Si `tono == "campesino"` o cualquier otro valor, retornar `SYSTEM_PROMPT_CAMPESINO`.
3. Si el valor no es ninguno de los dos reconocidos, registrar `logger.warning("[C2.1] Tono desconocido '{tono}', usando campesino")`.

### `build_contexto_recuperado(prediccion, narrativa, top_features, serie_historica, tono)`

1. Inicializar lista `partes = []`.
2. Si `prediccion` no es `None`:
   - Extraer `rendimiento_esperado` y `etiqueta_riesgo`.
   - Si `rendimiento_esperado` no es `None`, formatear según tono.
   - Si `etiqueta_riesgo` no es `None`, formatear semáforo según tono.
   - Si `prob_riesgo_alto` no es `None` y tono es `"institucional"`, incluirlo.
3. Si `narrativa` no es `None` y no es string vacío, agregar sección de análisis/explicación.
4. Si `top_features` no es `None` y tiene elementos:
   - Tomar máximo 3 features.
   - Para cada una, llamar a `format_feature_for_prompt(feat, tono)`.
5. Si `serie_historica` no es `None` y tiene elementos:
   - Tomar los últimos 3 registros con `rendimiento` no `None`.
   - Formatear según tono.
6. Si `partes` está vacío al final, retornar el string `"No hay información disponible para este contexto."`.
7. Retornar `"\n".join(partes)`.

### `format_feature_for_prompt(feature, tono)`

1. Extraer `nombre_amigable` o `feature_id` o `feature` como nombre (en ese orden de preferencia).
2. Si ninguna clave existe, usar `"factor desconocido"`.
3. Extraer `valor_original` o `valor`. Si no existe, usar `None`.
4. Extraer `direccion`. Si no existe, usar `""`.
5. Formatear según tono con las reglas actuales.

### `build_user_prompt(pregunta, contexto, contexto_recuperado, glosario_relevante)`

1. Si `pregunta` es `None` o vacía, usar `"Sin pregunta especificada"`.
2. Construir `contexto_str` a partir de los campos no nulos de `contexto`.
3. Si `contexto_recuperado` es `None` o vacío, usar `"No se encontró información específica."`.
4. Si `glosario_relevante` no es `None` y no es vacío, incluir sección de términos.
5. Retornar el prompt completo.

### `build_prompt_conversacional(contexto_recuperado, pregunta, tono)` ← función nueva

1. Llamar a `build_contexto_recuperado(...)` con los campos del objeto `contexto_recuperado`.
2. Llamar a `build_user_prompt(pregunta, contexto_recuperado.contexto_efectivo, texto_contexto, "")`.
3. Llamar a `build_system_prompt(tono)`.
4. Retornar `{"system": system_prompt, "user": user_prompt}` como `dict`.

## Manejo de errores

| Situación                                    | Comportamiento                                                                                      |
| -------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| `tono` no reconocido                         | `logger.warning("[C2.1] Tono desconocido '{tono}', usando campesino")` → usar `"campesino"`        |
| `pregunta` es `None` o vacía                 | Usar `"Sin pregunta especificada"` como placeholder, no lanzar excepción                            |
| `prediccion` es `None`                       | Omitir sección de predicción, no lanzar excepción                                                   |
| `feature` sin clave `nombre_amigable`        | Usar `feature_id` o `feature` o `"factor desconocido"` en ese orden                                |
| `contexto_recuperado` sin campos con valor   | Retornar `"No hay información disponible para este contexto."` como texto de contexto               |
| `serie_historica` con entradas sin rendimiento | Omitir esas entradas, no lanzar excepción                                                         |

## Salida esperada

- `build_system_prompt("campesino")` → string con el system prompt campesino (≥ 100 caracteres).
- `build_system_prompt("institucional")` → string con el system prompt institucional (≥ 100 caracteres).
- `build_contexto_recuperado(None, None, None, None)` → `"No hay información disponible para este contexto."`.
- `build_prompt_conversacional(ctx, "¿Cuál es el riesgo?", "campesino")` → `dict` con claves `"system"` y `"user"`, ambas strings no vacíos.

## Caso feliz

- Dado un `ContextoRecuperado` con `prediccion`, `narrativa` y `top_features` no nulos.
- Cuando se llama a `build_prompt_conversacional(ctx, "Explícame el riesgo", "campesino")`.
- Entonces retorna un `dict` con `"system"` y `"user"` no vacíos, donde `"user"` contiene
  el rendimiento esperado, la narrativa y al menos un factor SHAP formateado.

## Validación

- Dado un `ContextoRecuperado` completamente vacío (todos los campos `None`).
- Cuando se llama a `build_prompt_conversacional(ctx_vacio, "", "campesino")`.
- Entonces retorna un `dict` con `"system"` y `"user"` no vacíos, sin lanzar excepción,
  y `"user"` contiene el placeholder de contexto no disponible.

## Criterios de aceptación

1. Las cuatro funciones existentes mantienen su firma pública sin cambios.
2. `build_prompt_conversacional()` retorna `dict` con claves `"system"` y `"user"`.
3. Ninguna función lanza excepción cuando todos los campos opcionales son `None`.
4. El tono `"campesino"` usa lenguaje coloquial; el tono `"institucional"` usa lenguaje técnico.
5. Un tono desconocido produce un warning y usa `"campesino"` como fallback.
6. `build_contexto_recuperado()` con todos los campos `None` retorna el string de contexto vacío.
7. `format_feature_for_prompt()` con un dict vacío retorna un string no vacío sin lanzar excepción.
