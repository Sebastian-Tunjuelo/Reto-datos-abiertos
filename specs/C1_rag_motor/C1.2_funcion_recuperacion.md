## Agente

- role: Ingeniero de recuperación de información / RAG.
- goal: Implementar la función de recuperación que, dado un municipio y cultivo, devuelve los documentos más relevantes del índice con scoring y los empaqueta en `ContextoRecuperado`.
- backstory: La función actual en `rag.py` filtra por igualdad exacta y toma solo el registro más reciente. La nueva versión debe soportar búsqueda por año, ranking por relevancia y fallback gracioso cuando no hay coincidencia exacta.

## Objetivo

Refactorizar `recuperar_contexto()` en `modules/conversational/rag.py` para que use el `RagIndex`
construido en C1.1, aplique scoring de relevancia, soporte filtro por año y mantenga la firma pública
existente para no romper `chat_engine.py`.

## Alcance

### Incluye

- Refactorización de `recuperar_contexto()` para usar `get_rag_index()` en lugar de leer el Parquet en cada llamada.
- Función interna `_score_document(doc: RagDocument, año: int | None) -> float` que calcula relevancia.
- Selección del documento de mayor score cuando hay múltiples registros para el mismo `(codigo_dane, cultivo)`.
- Fallback: si no hay coincidencia exacta por `(codigo_dane, cultivo)`, retornar `ContextoRecuperado` vacío con warning.
- Mantener la firma pública de `recuperar_contexto()` sin cambios para no romper `chat_engine.py`.
- Mantener la clase `ContextoRecuperado` y su método `to_dict()` sin cambios.

### No incluye

- Búsqueda semántica por embeddings.
- Recuperación de documentos de otros Parquets (`feature_matrix.parquet`) — eso queda en la lógica existente de serie histórica.
- Cambios en `chat_engine.py`, `prompts.py` ni `reports.py`.
- Recuperación multi-municipio o multi-cultivo en una sola llamada.

## Entradas

- `RagIndex` construido por C1.1 (accedido vía `get_rag_index()`).
- Parámetros de la llamada: `municipio`, `cultivo`, `año`, `escenario`, `tono`.
- `shared/dane_codes.py` → `get_codigo()`, `DANE_TO_NAME`.
- `shared/normalization.py` → `normalize_cultivo()`, `normalize_dane_code()`.

## Inputs esperados

| Parámetro   | Tipo            | Regla                                                                                   |
| ----------- | --------------- | --------------------------------------------------------------------------------------- |
| `municipio` | `str \| None`   | Nombre display o código DANE. Si es dígitos, tratar como código; si no, resolver con `get_codigo()`. |
| `cultivo`   | `str \| None`   | Normalizar con `normalize_cultivo()`. Si no resuelve, retornar `ContextoRecuperado` vacío. |
| `año`       | `int \| None`   | Año de referencia para scoring. Si es `None`, priorizar el año más reciente del índice. |
| `escenario` | `str \| None`   | Informativo; se incluye en `contexto_efectivo` pero no afecta la recuperación del índice. |
| `tono`      | `str`           | `campesino` o `institucional`. Informativo; no afecta la recuperación.                  |

## Lógica / Proceso sugerido

1. Normalizar `municipio` a `codigo_dane` usando `get_codigo()` o `normalize_dane_code()`.
   - Si el municipio no se reconoce, loggear warning y retornar `ContextoRecuperado` vacío.
2. Normalizar `cultivo` con `normalize_cultivo()`.
   - Si el cultivo no se reconoce, loggear warning y retornar `ContextoRecuperado` vacío.
3. Obtener el índice con `get_rag_index()`.
4. Consultar `index.get(codigo_dane, cultivo)` para obtener la lista de documentos candidatos.
   - Si la lista está vacía, loggear warning `"[C1.2] Sin documentos para {municipio}/{cultivo}"` y retornar `ContextoRecuperado` vacío.
5. Calcular el score de cada documento con `_score_document(doc, año)`:
   - Si `año` es `None`: score = `doc.año` (priorizar el más reciente).
   - Si `año` es el mismo que `doc.año`: score = 1000 (coincidencia exacta).
   - Si `año > doc.año`: score = `1000 - (año - doc.año)` (penalizar por distancia temporal).
   - Si `año < doc.año`: score = `500 - (doc.año - año)` (penalizar más por ser futuro).
6. Seleccionar el documento con mayor score.
7. Construir `ContextoRecuperado` a partir del documento seleccionado:
   - `prediccion`: `{"rendimiento_esperado": doc.rendimiento_t1, "etiqueta_riesgo": doc.prediccion_riesgo}`.
   - `narrativa`: `doc.narrativa_riesgo` si no es `None`.
   - `top_features`: `doc.top_features` si no está vacío.
   - `fuentes`: lista con los artefactos efectivamente usados.
   - `contexto_efectivo`: `{municipio, cultivo, año, escenario}`.
8. Intentar cargar serie histórica desde `feature_matrix.parquet` (lógica existente, sin cambios).
9. Retornar el `ContextoRecuperado` completo.

## Manejo de errores

| Situación                                         | Acción                                                                                                  |
| ------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| Municipio no reconocido                           | `logger.warning("[C1.2] Municipio no reconocido: {municipio}")` — retornar `ContextoRecuperado` vacío  |
| Cultivo no normalizable                           | `logger.warning("[C1.2] Cultivo no reconocido: {cultivo}")` — retornar `ContextoRecuperado` vacío      |
| Sin documentos para `(codigo_dane, cultivo)`      | `logger.warning("[C1.2] Sin documentos para {municipio}/{cultivo}")` — retornar `ContextoRecuperado` vacío |
| Índice no disponible (error en C1.1)              | `logger.error("[C1.2] Índice RAG no disponible: {e}")` — retornar `ContextoRecuperado` vacío           |
| `feature_matrix.parquet` no encontrado            | `logger.warning("[C1.2] Feature matrix no encontrada")` — continuar sin serie histórica                |

## Salida esperada

- `ContextoRecuperado` con el documento más relevante para `(municipio, cultivo, año)`.
- `fuentes` lista los artefactos efectivamente usados: `["predicciones_con_explicacion.parquet", "narrativa_riesgo", "top_features_shap"]`.
- Si no hay contexto, retorna `ContextoRecuperado` vacío (no lanza excepción).
- La función es determinista: mismos inputs → mismo output.
