fuga de informacion.

**Mentalidad**: Piensa en contratos. El output debe tener 25 columnas exactas en
orden fijo. Cualquier columna prohibida es una falla critica.

**Restricciones**:

- No usar `inplace=True`
- No introducir columnas fuera de `COLUMNAS_FEATURE_MATRIX`
- No permitir columnas prohibidas: `produccion`, `area_cosechada`, `rendimiento`

## Logica de transformacion

### 1) `_map_aptitud_columns(df)`

```
- Renombrar columnas de aptitud D5 → M1
- Si faltan columnas de aptitud para un MVP, dejar NaN y loggear warning
- Verificar suma aptitud por (codigo_dane, cultivo)
  - Si difiere de 100 en >1 pp → warning
- Verificar suma frontera por codigo_dane (pct_condicionada + pct_no_condicionada)
  - Si difiere de 100 en >1 pp → warning
```

### 2) `_join_agroinsumos(df, data_dir)`

```
- Cargar agroinsumos.parquet (si no existe → FileNotFoundError)
- Left join por año
- indice_total → indice_agroinsumos
- Si un año no tiene datos → NaN en indice/percentil/señal y warning
```

### 3) `_calc_percentil_fert(df, data_dir)`

```
- Usar fertilizantes de agroinsumos.parquet
- Para cada año t: percentil del valor fertilizantes[t]
  dentro de los años <= t (expanding)
- Asignar a percentil_fertilizantes (0–100)
```

### 4) Validacion de señal de riesgo

```
- señal_riesgo_economico solo acepta {bajo, medio, alto}
- Cualquier otro valor → NaN + warning
```

### 5) `_rename_target(df)`

```
- rendimiento → target_rendimiento
- Preservar NaN
```

### 6) `_enforce_schema(df)`

```
- pd.to_numeric(..., errors='coerce') en columnas numericas
- Agregar columnas faltantes como NaN
- Seleccionar y ordenar COLUMNAS_FEATURE_MATRIX
```

### 7) `_assert_no_leakage(df)`

```
- Si alguna columna prohibida existe → AssertionError
```

### 8) `_validate_mvp(df)`

```
- Verificar presencia de los 15 MVP_CODIGOS (warning por cada ausente)
- En train (<=2021), features criticas sin NaN:
  rendimiento_t1, prec_acum_mm, temp_media_c, indice_agroinsumos
- rendimiento_t1 dentro de RENDIMIENTO_RANGOS por cultivo
- prec_acum_mm > 0 en train
- Suma aptitud en [99, 101] por (codigo_dane, cultivo) del MVP
- >= 3 filas en train por (codigo_dane, cultivo)
- Loggear resumen final PASS/WARN
```

### 9) `_save_and_log(df, output_dir)`

```
- Guardar feature_matrix.parquet con engine='pyarrow'
- logger.info con filas, columnas, municipios y cultivos
```

### 10) `build_feature_matrix(...)`

Encadena los pasos en este orden:

```
_validate_input → _apply_climate_ranges → _build_lag_features →
_map_aptitud_columns → _join_agroinsumos → _calc_percentil_fert →
_rename_target → _enforce_schema → _assert_no_leakage →
_validate_mvp → _save_and_log
```
