## Agente

- **role**: Ingeniero de integración LLM / módulo conversacional.
- **goal**: Agregar `build_reporte_umata_llm()` a `reports.py` para que el contenido narrativo del reporte sea generado por el LLM usando el prompt UMATA de C2.2, en lugar de texto estático.
- **backstory**: A3.3 generó reportes con texto fijo. C3.1 eleva la calidad narrativa usando Gemini para producir un análisis contextualizado, manteniendo compatibilidad total con `render_pdf()` y el endpoint existente.

---

## Objetivo

Agregar la función `build_reporte_umata_llm()` a `modules/conversational/reports.py`.
La función recupera contexto vía RAG (C1), construye el prompt UMATA (C2.2), llama al LLM
y retorna un dict con el mismo esquema que `build_reporte()`, con `contenido_texto` generado
por el LLM. Si el LLM falla, hace fallback a `build_reporte()`.

---

## Alcance

### Incluye

- Función `build_reporte_umata_llm(codigo_dane, municipio, departamento, cultivo, tono)` en `reports.py`.
- Llamada a `recuperar_contexto()` de `rag.py` para obtener `ContextoRecuperado`.
- Llamada a `build_prompt_reporte_umata()` de `prompts.py` para construir el prompt.
- Llamada al LLM vía `get_chat_engine()` de `chat_engine.py`.
- Fallback a `build_reporte()` si el LLM no está disponible o lanza excepción.
- Logging de cada etapa (contexto recuperado, prompt construido, respuesta LLM, fallback).

### No incluye

- Cambios en la firma de `build_reporte()` ni `render_pdf()`.
- Modificación del endpoint en `orchestrator/main.py`.
- Reentrenamiento de modelos ni recálculo de SHAP.
- Almacenamiento persistente del contenido generado.

---

## Entradas

- `codigo_dane`: str, 5 dígitos, código DANE del municipio.
- `municipio`: str, nombre display en Title Case.
- `departamento`: str, nombre del departamento en Title Case.
- `cultivo`: str, uno de `'Café'`, `'Cacao'`, `'Maíz'`.
- `tono`: str, `'institucional'` (default) o `'campesino'`.

## Lógica / Proceso sugerido

1. Llamar a `recuperar_contexto(municipio=municipio, cultivo=cultivo, tono=tono)` para obtener `ContextoRecuperado`.
2. Si `contexto.prediccion` es `None`, lanzar `ValueError` con mensaje descriptivo (sin predicción no se puede generar el reporte).
3. Extraer de `ContextoRecuperado`: `prediccion_riesgo`, `narrativa`, `top_features`, `serie_historica`.
4. Cargar `clima_row` directamente desde `clima_agregado.parquet` (igual que `build_reporte()`).
5. Construir el prompt llamando a `build_prompt_reporte_umata(...)` con los datos extraídos.
6. Llamar a `get_chat_engine().model.generate_content(prompt, ...)` con `max_output_tokens=800`, `temperature=0.4`.
7. Parsear la respuesta del LLM como `contenido_texto`.
8. Si el LLM lanza excepción, loggear `logger.warning("[C3.1] LLM no disponible, usando fallback")` y llamar a `build_reporte(codigo_dane, municipio, departamento, cultivo)` como fallback.
9. Construir y retornar el dict de reporte con el mismo esquema que `build_reporte()`, añadiendo la clave `"generado_por_llm": True` (o `False` si se usó fallback).

---

## Interfaz o contrato

```python
def build_reporte_umata_llm(
    codigo_dane: str,
    municipio: str,
    departamento: str,
    cultivo: str,
    tono: str = "institucional",
) -> dict:
    """
    Construye el reporte usando el LLM (Gemini) para generar el contenido narrativo.

    Retorna el mismo esquema que build_reporte() más la clave 'generado_por_llm'.
    Si el LLM no está disponible, hace fallback a build_reporte().

    Raises:
        ValueError: si no hay predicción disponible para el par municipio/cultivo.
        FileNotFoundError: si los parquets requeridos no existen.
    """
```

### Esquema de retorno

```python
{
    "codigo_dane": str,          # 5 dígitos
    "municipio": str,            # Title Case
    "departamento": str,         # Title Case
    "cultivo": str,              # 'Café' | 'Cacao' | 'Maíz'
    "año_referencia": int,
    "titulo": str,
    "contenido_texto": str,      # Generado por LLM o fallback estático
    "secciones": list[str],      # Mismo orden que SECCIONES_ORDEN
    "fuentes": list[str],
    "generado_por_llm": bool,    # True si el LLM respondió, False si se usó fallback
}
```

---

## Manejo de errores

| Situación                                    | Comportamiento                                                                 |
| -------------------------------------------- | ------------------------------------------------------------------------------ |
| `prediccion` es `None` en `ContextoRecuperado` | Lanzar `ValueError("No hay predicción disponible para {municipio} / {cultivo}")` |
| LLM lanza excepción (cuota, red, timeout)    | `logger.warning("[C3.1] LLM no disponible, usando fallback")` + llamar `build_reporte()` |
| `build_prompt_reporte_umata` no existe       | Lanzar `ImportError` con mensaje claro indicando que C2.2 debe implementarse  |
| Parquet no encontrado                        | Propagar `FileNotFoundError` (igual que `build_reporte()`)                     |
| Respuesta LLM vacía o solo whitespace        | Tratar como fallo de LLM → fallback a `build_reporte()`                        |

---

## Salida esperada

- `build_reporte_umata_llm("73168", "Chaparral", "Tolima", "Café")` retorna un dict con `contenido_texto` narrativo generado por Gemini, `generado_por_llm: True` y `fuentes` que incluye `"llm:gemini"`.
- Si Gemini no está disponible, retorna el mismo dict que `build_reporte()` con `generado_por_llm: False`.
- `render_pdf(resultado)` acepta el dict sin modificaciones.

## Validación

- **Dado** un municipio válido pero LLM con cuota agotada (excepción al llamar a Gemini).
- **Cuando** se llama a `build_reporte_umata_llm(...)`.
- **Entonces** retorna el dict de `build_reporte()` con `generado_por_llm=False` y no lanza excepción.

- **Dado** un municipio sin predicción en el parquet.
- **Cuando** se llama a `build_reporte_umata_llm(...)`.
- **Entonces** lanza `ValueError` con mensaje que incluye municipio y cultivo.

1. `build_reporte_umata_llm()` existe en `modules/conversational/reports.py` y es importable.
