## Agente

- role: Ingeniero frontend Next.js.
- goal: Entregar un mapa interactivo que muestre riesgo por municipio para un cultivo y anio objetivo.
- backstory: Es la pantalla principal del MVP y debe comunicar riesgo de forma clara a UMATAs y extensionistas.

## Objetivo

- Mostrar los 15 municipios del MVP en un mapa de Colombia, coloreados por `etiqueta_riesgo` para el cultivo y anio seleccionados.

## Alcance

### Incluye

- Render de mapa con 15 municipios del MVP (poligonos GeoJSON).
- Selector de cultivo y control de anio objetivo.
- Leyenda de colores y estados (Bajo, Medio, Alto, Sin datos).
- Tooltip o panel breve con `rendimiento_esperado`, `prob_riesgo_alto` y `etiqueta_riesgo`.
- Carga en paralelo de predicciones por municipio y cache local por combinacion `codigo_dane + cultivo + anio`.
- Estados de carga, error parcial y vacio.
- Diseno responsive para desktop y mobile.

### No incluye

- Ficha municipal detallada (F2).
- Comparador de cultivos (F3).
- Chat o reportes.
- Nuevos endpoints backend.
- Geocodificacion o busqueda global.

## Entradas

- `GET /municipios` (A1.1).
- `GET /cultivos/{municipio}` (A1.2) para poblar el selector sin hardcode.
- `POST /predecir` (A1.3) para riesgo por municipio.
- `public/geo/colombia_mvp.geojson` con 15 municipios.

## Inputs esperados

### API

- `GET /municipios` -> lista de `{ codigo_dane, municipio, departamento }`.
- `GET /cultivos/{municipio}` -> `{ codigo_dane, municipio, departamento, cultivos }`.
- `POST /predecir` -> `{ codigo_dane, municipio, departamento, cultivo, anio, rendimiento_esperado, prob_riesgo_alto, etiqueta_riesgo }`.

### GeoJSON

- Archivo GeoJSON en `frontend/public/geo/colombia_mvp.geojson`.
- Cada feature debe tener:
  - `properties.codigo_dane` (string, 5 digitos).
  - `properties.municipio` (string, Title Case).
- Coordenadas en WGS84 (EPSG:4326).

## Logica / Proceso sugerido

1. Cargar el catalogo con `GET /municipios` y construir un mapa `codigo_dane -> { municipio, departamento }`.
2. Cargar el GeoJSON y filtrar features por los 15 `codigo_dane` del catalogo.
3. Resolver cultivo inicial con `GET /cultivos/{primer_municipio}`; si falla, usar el primer valor disponible en `CULTIVOS_MVP` retornado por el backend.
4. Para cada municipio, invocar `POST /predecir` con `{ municipio: codigo_dane, cultivo, anio }` usando concurrencia acotada.
5. Guardar resultados en cache local por `codigo_dane + cultivo + anio` para evitar refetch innecesario.
6. Renderizar poligonos con color segun `etiqueta_riesgo`:
   - Bajo: verde
   - Medio: amarillo
   - Alto: rojo
   - Sin datos / error: gris
7. Al hover o click, mostrar resumen con municipio, cultivo, anio, `rendimiento_esperado` y `prob_riesgo_alto`.
8. Si el usuario cambia cultivo o anio, reusar cache y solo pedir lo faltante.

## Interfaz o contrato

### Pagina

- Ruta `/`.
- Componente principal: `MapaColombia` (client component con `react-leaflet`).

### Estado en UI (shape sugerido)

```ts
type MapaMunicipio = {
  codigo_dane: string;
  municipio: string;
  departamento: string;
  cultivo: string;
  anio: number;
  rendimiento_esperado: number | null;
  prob_riesgo_alto: number | null;
  etiqueta_riesgo: "Bajo" | "Medio" | "Alto" | "Sin datos";
  estado: "ok" | "sin_datos" | "error" | "cargando";
};
```

## Manejo de errores

| Situacion                            | Comportamiento UI                                                |
| ------------------------------------ | ---------------------------------------------------------------- |
| `POST /predecir` 404 (sin datos)     | Marcar municipio como `Sin datos`, color gris.                   |
| `POST /predecir` 422 (anio invalido) | Mostrar mensaje y mantener ultimo estado valido.                |
| Timeout o 5xx en algunas llamadas    | Mostrar banner de error parcial y permitir reintento.           |
| GeoJSON no disponible                | Mostrar estado de error bloqueante con mensaje accionable.      |

## Salida esperada

- Mapa interactivo con 15 municipios coloreados por riesgo.
- Selector de cultivo y anio funcionales.
- Leyenda visible y tooltips con valores clave.

## Caso feliz

- Dado el catalogo de municipios y el GeoJSON cargados.
- Cuando el usuario selecciona un cultivo y anio.
- Entonces el mapa muestra los 15 municipios con color segun `etiqueta_riesgo`.

## Validacion

- Dado un anio no valido.
- Cuando se intenta consultar.
- Entonces la UI muestra un error y no rompe el mapa.

## Criterios de aceptacion

- El mapa usa `codigo_dane` como llave de union entre GeoJSON y API.
- No hay codigos DANE hardcodeados en el frontend.
- Cambiar cultivo o anio actualiza el color de los 15 municipios.
- El mapa funciona en mobile y desktop.
- La UI mantiene legibilidad y contraste en los 4 estados de color.

## Restricciones tecnicas

- Next.js 14 App Router.
- `react-leaflet` debe cargarse solo en cliente (sin SSR).
- Usar `NEXT_PUBLIC_API_BASE_URL` para configurar el backend.
- No usar Mapbox ni servicios pagos.
- Fuente tipografica no puede ser Inter, Roboto, Arial ni system.
- Evitar estilo plano: definir variables CSS de color y fondo con gradiente sutil.
- Animaciones: 1 carga inicial y 1 transicion suave al cambiar cultivo o anio.
